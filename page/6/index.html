Java ArrayList 数组大小动态调整

發表於 2017-11-06   |   分類於 blog

在向ArrayList中添加元素时，都调用了ensureCapacityInternal（）方法，该方法用于调整数组大小。过程如下：
每次调用add操作时，都会对size进行加1操作，然后调用ensureCapacityInternal获得两者较大的那一个，当数组满了的时候，会出现size+1>DEFAULT_CAPACITY的情况，这时候也就需要重新调整数组大小，调用grow进行调整。
grow方法中，数组大小进行调整时，会默认扩大为原来的1.5倍，newCapacity = oldCapacity + (oldCapacity >> 1),然后将newCapacity与size+1进行比较，取两者中较大的一个；作为最终的大小。
如果newCapacity的值超过了VM所允许的最大数组分配个数，就调用hugeCapacity获取最终数组的大小。
最后，调用Arrays.copy,将原先数组中的元素copy到一个更大容量的数组上，然后返回这个新的数组

/**
 *确定数组的容量：
 *如果数组为空，则容量为默认的DEFAULT_CAPACITY--10
 *否则，容量为minCapacity
 */
private void ensureCapacityInternal(int minCapacity) {
        if (elementData == EMPTY_ELEMENTDATA) {
            minCapacity = Math.max(DEFAULT_CAPACITY, minCapacity);
        }

        ensureExplicitCapacity(minCapacity);
    }

/**
 *修改次数加1
 *如果最小容量大于数组当前的容量，调用grow()，进行大小调整
 */

private void ensureExplicitCapacity(int minCapacity) {
        modCount++;

        // overflow-conscious code
        if (minCapacity - elementData.length > 0)
            grow(minCapacity);
    }

/**
     * The maximum size of array to allocate.
     * Some VMs reserve some header words in an array.
     * Attempts to allocate larger arrays may result in
     * OutOfMemoryError: Requested array size exceeds VM limit
     */
    private static final int MAX_ARRAY_SIZE = Integer.MAX_VALUE - 8;

    /**
     * Increases the capacity to ensure that it can hold at least the
     * number of elements specified by the minimum capacity argument.
     *
     * @param minCapacity the desired minimum capacity
     */
    private void grow(int minCapacity) {
        // overflow-conscious code
        int oldCapacity = elementData.length;
        //默认扩展0.5倍
        int newCapacity = oldCapacity + (oldCapacity >> 1);
        if (newCapacity - minCapacity < 0)
            newCapacity = minCapacity;
        if (newCapacity - MAX_ARRAY_SIZE > 0)
            newCapacity = hugeCapacity(minCapacity);
        // minCapacity is usually close to size, so this is a win:
        elementData = Arrays.copyOf(elementData, newCapacity);
    }

    private static int hugeCapacity(int minCapacity) {
        if (minCapacity < 0) // overflow
            throw new OutOfMemoryError();
        return (minCapacity > MAX_ARRAY_SIZE) ?
            Integer.MAX_VALUE :
            MAX_ARRAY_SIZE;
    }

java异常分类

![此处输入图片的描述][1]

  • 如上图，所有的异常类继承自Throwable。
  • 在第二层分为两部分：Error和Exception
  • Error:指java运行时系统内部错误和资源耗尽错误，应用程序不应该抛出这种类型的对象。当出现这种错误时，应该通知给用户，并尽力使程序安全终止。
  • Exception:分为两类：RuntimeException和IOException.划分这两种异常的规则是：由程序错误导致的异常属于RuntimeException;而程序没有错误，由于像I/O错误导致的异常属于其他异常
  • java语言规范将派生于Error类或RuntimeException类的所有异常称为非受查（unchecked）异常；所有其他的异常称为checked异常。
  • RuntimeException通常以下几种情况：
    • 错误的类型转换
    • 数组访问越界
    • 访问null指针
  • 非RuntimeException的异常包括：
    • 访问一个不存在的文件
    • 试图在文件尾部后面读取数据
    • 试图根据给定的字符串查找Class对象，而这个字符串表示的类并不存在

“如果出现RuntimeException异常，那么就一定是你的问题”

###异常的处理

  • 声明受查异常（如果一个方法可能抛出的多个受查异常类型，那么就必须在方法首部列出所有的异常类，每个异常类之间用逗号隔开）
  • 不需要声明java的内部错误，即从Error继承的错误
  • 不需要声明从RuntimeException继承的非受查异常

###如何抛出一个异常

  • 找到一个合适的异常类，并声明该异常
  • 创建这个类的一个对象
  • 将对象抛出

###创建一个异常类

  • 继承Exception或Exception子类
  • 带有两个构造器，一个是默认的构造器；一个是带有详细描述信息的构造器
  • 示例代码如下
class FileFormatException extends Exception{
    public FileFormatException(){}
    public FileFormatException(String gripe){
    super(gripe)
    }
}

###捕获异常
使用try/catch语句块捕捉异常

###finally语句：不管是否有异常被捕获，finally字句中的代码都被执行
示例代码

InputStream in= new FileInputStream(...)
try{
//1
可能抛出异常的代码
//2
}
catch (IOException e){
//3
show error message
//4
}
finally{
//5
in.close
}
//6

可能的代码执行情况：

  • 代码没有抛出异常。执行1、2、5、6
  • 抛出一个在cache子句中捕获的异常，执行1、3、4、5、6
  • 抛出一个异常，但是该异常没有被catch语句捕捉，则执行finally语句，并将异常抛给这个方法的调用者，finally语句后的6并不会执行，这里执行1、5处的语句

####带有return语句的finally：
假设利用return语句从try语句块中退出。在方法返回前，finally子句中等待内容将被执行。如果finally字句中也有一个return语句，这个返回值将会覆盖原始的返回值。示例代码如下

public static int f(int n){
try{
int r = n * n;
return r;
}
finally{
if (n==2)return 0;
}
}
}

如果调用f（2），try语句块中计算结果为4，并执行return语句。然后再执行finally字句，将返回0，覆盖掉原始返回值4.

###带资源的try语句
当在finally语句中进行资源关闭操作时，若close方法抛出异常，这时候就会对close抛出的异常进行处理，代码会非常繁琐。这时可使用带资源的try语句。
示例如下：

try（Scanner in = new Scanner(new FileInputStream("/usr/share/dict/words")),"UTF-8"）
{
while(in.hasNext())
    System.out.println(in.next)
}

这个块在正常退出或存在异常时，会调用in.close()方法，就好像使用了finally块一样。
只需要关闭资源，就要尽可能使用带资源的try语句

DNS查询主机名IP的流程

發表於 2017-10-29   |   分類於 blog

###IP与主机名的历史

  • IP是由32位二进制（IPV4）组成，为了适应人们的习惯，将其转换成四组十进制的数字，如12.34.56.78
  • 由于IP数字这个格式不便于记忆，人们想到了一个方法，在一个文件中记录ip与主机名的对应关系，这个文件就是/etc/hosts，其内容为“IP主机名1主机名2”
  • 当IP数量过多时，导致/etc/hosts文件过大，不利于其他主机的同步。为了解决这个日益严重的问题，伯克利大学发明了域名系统（Domain Name System,DNS）.通过DNS，不需要知道主机的IP，只要知道该主机的名称，就能连上该主机。

###主机名与域名
拿电话号码来解释主机名与域名的关系。
假设有北京、上海各有一个电话号码1234567，那么如何区别这两个号码呢？用区号（区号对应的是域名），假设我想打给上海的1234567，就拨0211234567，同理，打给北京的就拨0101234567。这里021、010就是域名，而1234567是主机名。可见，通过域名来区分相同的主机名。
对于网站，www.google.com,www.seednet.net,www是主机名，那么怎么区分这两个主机名相同的机器呢？答案是通过域名，这里域名是google.com,seednet.net.所以即使两台机器主机名相同，位于不同的域中，也能够被区分出来。

###DNS的阶层架构

如图：
> 最上方是root服务器
> 第二层是顶级域名，如.com,.gov,.edu
> * 第三层是地区级域名，如.cn,.jp

###通过DNS查询主机名IP的流程。

  • 在浏览器中输入http://www.tsinghua.edu.cn时，计算机会根据/etc/resolv.conf这个文件提供的DNS的IP进行连接；
  • DNS服务器在收到用户请求后，先查询本身有没有这个记录，如果没有则向.(root)查询
  • DNS会询问www.tsinghua.edu.cn在哪里，.(root)只记录了.cn信息，则告诉去寻找.cn
  • 同样.cn记录了.edu.cn,.com.cn,.gov.cn,对比后发现，应该去.edu.cn那里寻找，所以告知.edu.cn记录
  • .edu.cn告知应该去tsinghua.edu.cn寻找
  • 在找到tsinghua.edu.cn后发现www这台主机确实是它管理的，则告知www.tsinghua.edu.cn的IP
  • 记录缓存并回报用户

并发与并行

  • 并行：是指在两个或多个进程同时运行，比如在每个处理器上运行一个进程
  • 并发：在同一个处理器上运行多个进程。通常使用时间片轮转的技术来实现。实际上，这种是伪并行，只是时间片很短，进程切换的非常快，看起来像是并行。
    并发是现代操作系统常用的技术。因为这种技术与计算机的处理器个数无关，所以操作系需要同时运行多个系统任务，而且还要满足用户任何的需求。为了解决这类问题，操作系统首先要关注处理器的任务调度时间表，记录每个任务需要的运行时间，然后在不同任务之间进行上下文切换，给每个任务一个时间片

多线程的优缺点

优点：

  • 持续响应：在单线程的程序中，执行一个长期运行的任务可能会导致程序冻结。多线程可以把这个长期运行的任务放在一个工作线程（worker thread）中，在程序并发地运行任务时可以持续响应客户需求。
  • 更快的响应速度：在多核或多处理器的操作系统上，多线程可以通过真正的并行提高程序运行速度
  • 较低的资源消耗：利用线程模式，程序可以利用一个进程的资源响应多个请求
  • 更简单的状态共享与进程间通信机制：由于线程都共享同一资源和内存空间，因此线程之间的通信比进程间通信更简单
  • 并行化：多核与多处理器系统可以实现多线程的每个线程独立运行。

缺点：

  • 线程同步
  • 某个线程出现问题导致集体崩溃
  • 死锁

GLI与多线程的使用场景

python的官方版本（CPython）有一个GIL限制。GIL会阻止多线程同时运行Python的字节码，所以这就不是真正的并发。CPython的GIL是有必要的，因为python的内存管理不是线程安全的。因此，为了让每个任务都按顺序进行，它需要保证运行过程中内存不受干扰。
（有一种说法是GIL只是CPython（python编译器）的产物，而与python语言本身没有关系）

基于GIL的原因，不应该使用Python线程来处理计算密集型的任务，因为这种任务更适合在多个CPU核心上实现并行处理。python线程更适合于I/O处理以及涉及阻塞操作的并发执行任务，如等待I/O，等待从数据库取出结果等。

python中多线程的实现方式

python中有两种方式实现多线程：

  • thread模块：这个模块提供了有限的线程能力。它很容易使用，适合小项目，不过会增加一点额外的资源消耗
  • threading模块：这是从Python2.4引入的新模块，提供了一些更强大、更高层次的线程支持。

python普遍推荐使用threading模块。

多线程：可以并发检查所有网站的状态，不需要在检查完成之后再去运行另一个检查
代码如下：

import urllib3
import time
from threading import Thread
class GetUrlThread(Thread):

    def __init__(self,url):
        self.url = url
        super(GetUrlThread, self).__init__()

    def run(self):
        http = urllib3.PoolManager()
        response = http.request('GET',self.url)
        print(self.url,response.status)

def getResponse():
    urls = [
            'http://www.appleieye.com',
            'http://www.amazon.com',
            'http://www.ebay.com',
            # 'http://www.alibaba.com',
            'http://www.reddit.com'
        ]
    start = time.time()
    threads = []
    for url in urls:
        thread = GetUrlThread(url)
        threads.append(thread)
        thread.start()

    for t in threads:
        t.join()

    print('time spent:{}'.format(time.time()-start))
    #所耗费的时间为7.181410789489746。

对比两种实现，可见多线程效率几乎是单线程的4倍。

对临界区加锁：

考虑一个例子：用多线程轮流对一个全局变量globalValue进行加一操作，直至加到200（数字越大，线程运行时的不确定性才能表现出来）

import threading

globalValue = 0
class ShareCounter(threading.Thread):

    def __init__(self,name):
        threading.Thread.__init__(self)
        self.name = name

    def run(self):
        global globalValue
        localCopy = globalValue
        print("{} with value {}".format(self.name,localCopy))
        globalValue = localCopy + 1

if __name__ == '__main__':
    for index in range(200):
        t = ShareCounter("thread"+str(index))
        t.start()

部分打印结果如下：


由打印结果看出，线程126/127/128/130/134打印的结果都是一样的，并且数字输出的顺序也会有错乱，最终得到的数值也没有到达200。
这是由于线程执行的速度、顺序不确定造成的。所以，需要确保代码在复制数值、进行加一操作以及打印数值的时候是处于加锁状态，没有两个线程来同时访问该数值。这时候就用到了锁：加锁和释放锁。
代码如下：

_lock = threading.RLock()
def run(self):
    ShareCounter._lock.acquire()
    global globalValue
    localCopy = globalValue
    print("{} with value {}".format(self.name,localCopy))
    globalValue = localCopy + 1
    ShareCounter._lock.release()

这时候在运行这段代码，就会发现数值按顺序打印，并且没有重复值出现。

###pyc文件：
PyCodeObject对象中都包含了每一个Code Block中所有python源代码经过编译后得到的byte code序列，python会将这些字节码序列和PyCodeObject对象一起存储在pyc文件中。但是并不是每次都会生成pyc文件。

  • 不会生成pyc的情况：某些python程序只是临时完成一些琐碎的工作，可能仅仅运行一次，然后就再也没有用了，所以没必要保存其对应的pyc文件
  • 需要生成pyc文件的情况：若某个程序需要被重用，比如通过import语句被加载，这时候就需要存储其对应的PyCodeObject对象，生成pyc文件，这样下次python就不会再次进行编译了。这意味着Python的import机制会触发pyc文件的生成，实际上，在python运行过程中，如果碰到import abc这样的语句，python将到设定好的path中寻找abc.pyc或abc.dll文件，如果没有这些文件，而只是发现了abc.py文件，那么python会首先将abc.py编译成相应的PyCodeObject的中间结果，然后创建abc.pyc文件，并将中间结果写入该文件。jiaxial接下来，python会对abc.pyc进行import的动作。实际上也就是将abc.pyc文件中的PyCodeObject重新在内存中复制出来。

###pyc文件生成过程

从write_compiled_module中可以发现，一个pyc文件中实际上包含了三部分独立的信息：magic number、pyc文件创建的时间信息，以及PyCodeObject对象。
过程如下：

  • 首先，python会将pyc_magic这个值写入文件开头。实际上，pyc_magic是python所定义的这个整数值，一般来说，不同版本的Python实现都会定义不同的magic number，用来保证python兼容性。比如说要要防止python2.5的环境加载由python1.5产生的pyc文件，那么只需要将python2.5和python1.5的magic number设为不同的值就可以了，因为python在加载pyc文件时会首先检查这个magic number，如果发现python自身的magic number与待加载的pyc文件中记录的magic number不同，则会拒绝加载不兼容的pyc文件
  • 写入pyc文件创建的时间信息。在pyc文件中包含时间信息可以使python自动将pyc文件与最新的py文件进行同步。在加载pyc文件中，如果python发现pyc文件的时间早于py文件，会重新编译py文件，生成新的pyc。
  • 将内存中的PyCodeObject对象写入pyc文件。

在调用PyMarshl_WriteLongToFile时，会直接调用w_long,w_long会将需要写入的数据一个字节一个字节地写入到文件中。而在调用PyMarshal_WriteObjectToFile时，python会借助于另一个辅助的函数w_object来完成将PyCodeObject对象写入到pyc文件中的操作。PyMarshal_WriteObjectToFile的第一个参数是python编译出来的PyCodeObject对象。w_object对应不同的对象，比如string、int、list、dict等，会有不同的写操作，最终是通过基本的w_long、w_string将整个PyCodeObject写入到pyc文件中去的。

举个例子，python在向pyc写入list对象时，只是将list中所包含的数值或字符串写入pyc文件。同时也意味着，python在加载pyc文件时，必须基于这些数值或字符串重新构造出list对象。这就要求在w_object写入对象之前，先写入TYPE_LIST、TYPE_CODE或者TYPE_INT这样的标识。这些标识对于pyc文件的再次加载具有至关重要的作用。
在前面说过，python最终仅仅会将数值和字符串写入到pyc文件中。当PyCodeObject对象写入到pyc文件之后，所有的数据都变成了字节流，类型信息都丢失了。如果没有类型信息，python再次加载pyc文件时，就再也没有办法知道这些字节流隐藏的结构和蕴含的信息了，所以Python必须将对象的类型信息也写入到pyc文件中，这些标识正是python定义的类型信息，如果python在pyc文件中发现 这样的标识，则预示着上一个对象结束，新的对象开始，而且也知道了新对象是什么类型的对象。只有这样，当python加载pyc文件时，加载器才能知道在什么时候应该进行什么样的加载操作。

  • python有一个非常核心的东西，这个东西叫作解释器（interpreter），我们在敲下python时，实际上激活了这个解释器。当我们通过python program.py执行一个特定的python程序时，python解释器立即被激活，然后执行Python程序。但是，在真正开始执行之前，python解释器还要完成一个非常复杂的工作–编译.py文件
  • python解释器在执行一个python程序文件时，首先进行的动作都是先对文件中的python源代码进编译，编译的主要结果是产生一组Python的byte code（字节码）,然后将编译的结果交给python的虚拟机，由虚拟机按照顺序一条一条地执行字节码，从而完成对python程序的操作。
  • python的编译器和虚拟机都存在于python25.dll
