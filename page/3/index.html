Java对象与类

类

对象：三个主要特性：行为、状态、标识

类之间的关系：依赖（use a）、聚合（has a）、继承（is a）.应该将相互依赖的类减至最少，即让类之间的耦合度最小

隐式参数与显式参数：隐式参数是出现在方法名前面的某个类的对象；显式参数是位于方法名后面括号中的值

方法参数：Java采用按值调用。

方法得到的是参数值的一个拷贝，方法不能修改传递给它的任何参数变量的内容。
举例如下：

public static void tripleValue(double x)
    {
     x = 3 * ;

    }
//调用该方法
double percent = 10;
tripleValue(percent);

执行完后，发现percent的值还是为10，具体执行过程如下:

  • x被初始化为percent的一个copy，也就是10
  • x被乘以3后等于30，但percent仍然是10
  • 该方法结束后，x不被使用
  • percent仍为10

方法参数共有两种类型：基本数据类型；对象引用。在上面例子中，已经看到了一个方法不能改变一个基本数据类型的参数，那么对象引用类型的参数呢？

public static void tripleSalary(Employee x)
{
 x.raiseSalary(200);
} 
当调用：
harry = new Employee();
tripleSalary(harry);

具体过程如下:

  • x被初始化为harry值的一个copy，这里是同一个对象的引用。
  • raiseSalary应用于这个对象的引用。x和harry同时引用的那个Employee对象的salary改变
  • 方法结束后，参数x不被使用，对象变量harry继续引用salary改变的对象

参数得到是对象引用的copy，对象引用以及他的copy同时引用同一个对象。

java中的引用不是按引用调用，反例如下:

public staic void swap(Employee x,Employee y)
 {
   Employee temp = x;
   x =  y;
   y = temp；
 }

如果java对对象采用的是按引用调用，那么上面的方法就能实现交换数据的效果，但是实际上并没有改变存储在变量a和b中的的对象引用。swap方法的参数x和y被初始化为两个对象应用的copy，这个方法交换的是两个copy

总结：

  • 一个方法不能修改一个基本数据类型的参数
  • 一个方法可以改变一个对象参数的状态
  • 一个方法不能让对象参数引用一个新的对象

重载:如果有多个方法由相同的名字、不同的参数，便产生了重载。方法名以及参数类型，叫作方法的签名。返回类型不是方法签名的一部分。也就是说不能有两个名字相同，参数类型也相同，但是返回不同类型值的方法。

静态方法

静态方法是一种不能向对象实施操作的方法。静态方法不能访问成员变量，因为它不能操作对象。
在下面两种情况使用静态方法：

  • 一个方法不需要访问对象状态，其所需参数都是通过显式参数提供
  • 一个方法只需要访问类的静态域

初始化块

初始化数据的方法：

  • 在构造器中赋值
  • 在声明中赋值
  • 初始化块：首先运行初始化块，然后才运行构造器的主体部分。

调用构造器的处理步骤;

  • 所有数据被初始化默认值（0，null或false）
  • 按照在类声明中出现的次序，依次执行所有域初始化语句和初始化块
  • 如果构造器中第一行调用了第二个构造器，则执行第二个构造器主体
  • 执行这个构造器主体

对静态变量的初始化：静态的初始化块，在类第一次加载的时候，会进行静态域的初始化，所有静态初始化语句以及静态初始化块都依照类定义的顺序执行。

类设计技巧

  • 要保证成员变量私有
  • 要对成员变量初始化
  • 不要在类中使用过多的基本类型，可用其他类代替多个相关的基本类型的使用
  • 将职责过多的类进行分解
  • 不是所有的域都需要独立的get和set方法
  • 类名和方法名要能体现他们的职责
  • 优先使用不可的类

其他

  • 一个对象变量并没有实际包含一个对象，而仅仅引用一个对象
  • 在java中，任何对象变量的值都是对存储在另外一个地方的一个对象的引用。new 操作符的返回值也是一个引用
  • 在一个源文件中，只能有一个public类，但可以有任意数目的非公有类

###为什么需要Django执行SQL
使用Django自带的ORM机制做查询有时会满足不了我们的需求， 例如某些查询没有对应的model；或者只想执行UPDATE、INSERT、或DELETE语句。在这些情况下，我们需要直接访问数据库，并想绕过model层。那么，我们应该如何做呢？
django.db.connection表示当前默认数据库的连接。如果想使用该连接，调用connection.cursor()来获得一个cursor对象；接着，调用cursor.execute(sql, [params])来执行SQL语句；然后使用cursor.fetchone()或cursor.fetchall()来返回查询结果。
示例如下：
from django.db import connection

def my_custom_sql(self):
    with connection.cursor() as cursor:
        cursor.execute("UPDATE bar SET foo = 1 WHERE baz = %s", [self.baz])
        cursor.execute("SELECT foo FROM bar WHERE baz = %s", [self.baz])
        row = cursor.fetchone()
    return row

注意：如果想在查询语句中包含百分号，必须写两个，示例如下

cursor.execute("SELECT foo FROM bar WHERE baz = '30%'")
cursor.execute("SELECT foo FROM bar WHERE baz = '30%%' AND id = %s", [self.id])

如果正在使用多个数据库，可以使用django.db.connections来获取指定数据库的连接（和cursor）。 django.db.connections是一个类似字典的对象，可以使用别名来获取指定的连接。

from django.db import connections
cursor = connections['my_db_alias'].cursor()
# Your code here...

默认情况下，Python DB API返回的结果中不包含字段名，这表示最终结果是值的list，而不是dict。可以使用以下代码返回一个dict。

def dictfetchall(cursor):
    "Return all rows from a cursor as a dict"
    columns = [col[0] for col in cursor.description]
    return [
        dict(zip(columns, row))
        for row in cursor.fetchall()
    ]

另一个选择是使用python标准库 collections.namedtuple()。namedtuple是类似tuple的对象，支持属性查询来访问字段；同时也支持索引和迭代。结果是不可变的，并且可通过字段名或切片访问，这两个特性可能会比较有用。

from collections import namedtuple

def namedtuplefetchall(cursor):
"Return all rows from a cursor as a namedtuple"
    desc = cursor.description
    nt_result = namedtuple('Result', [col[0] for col in desc])
    return [nt_result(*row) for row in cursor.fetchall()]

以上三种情况的差别如下所示：

>>> cursor.execute("SELECT id, parent_id FROM test LIMIT 2");
>>> cursor.fetchall()
((54360982, None), (54360880, None))

>>> cursor.execute("SELECT id, parent_id FROM test LIMIT 2");
>>> dictfetchall(cursor)
{'parent_id': None, 'id': 54360982}, {'parent_id': None, 'id': 54360880}]

>>> cursor.execute("SELECT id, parent_id FROM test LIMIT 2");
>>> results = namedtuplefetchall(cursor)
>>> results
[Result(id=54360982, parent_id=None), Result(id=54360880, parent_id=None)]
>>> results[0].id
54360982
>>> results[0][0]
54360982

###Connections 和 cursors
connection和cursor在大多数情况下都实现了PEP 249所描述的Python DB-API，除了事务处理。
如果对python DB-API不熟悉的话，要注意到cursor.execute()中的SQL语句使用了占位符”%s”,而不是SQL内直接添加参数。如果你使用这种机制，底层的数据库类库必要时将自动对参数进行转义。
同时也要注意，Djano使用”%s”作为占位符，而SQLite使用”?”做占位符。
将cursor用作上下文管理器

with connection.cursor() as c:
    c.execute(...)

等同于

c = connection.cursor()
    try:
        c.execute(...)
    finally:
        c.close()

###调用存储过程
CursorWrapper.callproc(procname, params=None)
使用给定的名称或可选的输入参数来调用存储过程
例如，Oracle的一个存储过程如下

CREATE PROCEDURE "TEST_PROCEDURE"(v_i INTEGER, v_text NVARCHAR2(10)) AS
    p_i INTEGER;
    p_text NVARCHAR2(10);
BEGIN
    p_i := v_i;
    p_text := v_text;
    ...
END;

使用如下方式调用：

with connection.cursor() as cursor:
cursor.callproc('test_procedure', [1, 'test'])

Iterator接口及其在ArrayList中的实现

###继承关系如下图所示

Itr类分析

####成员变量

  • cursor:下一个返回元素的序号
  • lastRet:上次返回的元素的序号
  • expectedModCount = modCount,修改次数

####方法

  • 判断是否还有下一个元素，依据：元素的index是否等于最后一个元素的index

public boolean hasNext() {
return cursor != size;
}

  • 每个迭代器检查自己改写操作的计数值是否与集合的改写操作计数值一致。如果不一致，则抛出ConcurrentModificationException
final void checkForComodification() {
       if (modCount != expectedModCount)
           throw new ConcurrentModificationException();
   }
  • 返回cursor指向的元素，然后cursor加1.这个方法会迷惑人，从字面意义上觉得是返回下一个元素，但事实却是返回当前元素，然后将cursor指向下一个元素。所以，next可以理解为将cursor指向next。
public E next() {
        checkForComodification();
        int i = cursor;
        if (i >= size)
            throw new NoSuchElementException();
        Object[] elementData = ArrayList.this.elementData;
        if (i >= elementData.length)
            throw new ConcurrentModificationException();
        cursor = i + 1;
        return (E) elementData[lastRet = i];
    }
  • remove方法：在调用remove方法之前，如果没有调用next方法，将会抛出IllegalStateException,原因是lastRet初始值为-1，小于0，抛出IllegalStateException;而调用next方法后，将lastRet赋值为cursor，即当前访问的元素，这样lastRet大于或等于0，不会抛出异常。然后调用ArrayList的remove方法删除元素；又将lastRest复位为-1，所以下次remove时，需要再次调用next
public void remove() {
        if (lastRet < 0)
            throw new IllegalStateException();
        checkForComodification();

        try {
            ArrayList.this.remove(lastRet);
            cursor = lastRet;
            lastRet = -1;
            expectedModCount = modCount;
        } catch (IndexOutOfBoundsException ex) {
            throw new ConcurrentModificationException();
        }
    }
  • forEachRemaining方法，java8中的新特性。以后补充。

###ListItr类分析：继承了Itr类，实现了ListIterator接口
方法如下：

  • 构造方法：将cursor初始化为index，默认值为0
ListItr(int index) {
    super();
    cursor = index;
}
  • 判断是否有前一个元素：依据：cursor是否等于0
public boolean hasPrevious() {
       return cursor != 0;
   }
  • 获取下一个元素的序号(cursor指向下一个元素的序号)
public int nextIndex() {
       return cursor;
   }
  • 获取上一个元素的序号(cursor指向下一个元素的序号)
public int previousIndex() {
       return cursor - 1;
   }
  • 获取前一个元素：如果当前元素(序号为cursor-1)为第一个元素，则抛出NoSuchElementException；将cursor指向当前元素(正常情况指向下一个元素)，返回当前元素
public E previous() {
        checkForComodification();
        int i = cursor - 1;
        if (i < 0)
            throw new NoSuchElementException();
        Object[] elementData = ArrayList.this.elementData;
        if (i >= elementData.length)
            throw new ConcurrentModificationException();
        cursor = i;
        return (E) elementData[lastRet = i];
    }
  • 用元素e取代next或previous返回的元素:在调用该方法前，必须先调用previous或next方法。否则,lastRet为-1，小于0，抛出异常。set方法并没有导致modCount加1操作，所以set方法不被视为结构性修改
public void set(E e) {
        if (lastRet < 0)
            throw new IllegalStateException();
        checkForComodification();

        try {
            ArrayList.this.set(lastRet, e);
        } catch (IndexOutOfBoundsException ex) {
            throw new ConcurrentModificationException();
        }
    }
  • 添加元素:ArrayList的add方法导致modCount次数加1，所以此操作会修改结构。当一个iter对集合进行遍历，而另一个对其修改时，则会抛出ConcurrentModificationException
public void add(E e) {
     checkForComodification();

     try {
         int i = cursor;
         ArrayList.this.add(i, e);
         cursor = i + 1;
         lastRet = -1;
         expectedModCount = modCount;
     } catch (IndexOutOfBoundsException ex) {
         throw new ConcurrentModificationException();
     }
 }

ArrayList继承的类和实现的接口


如上图所示，实现的接口有List、RandomAccess、Cloneable、java.io.Serializable;继承的类有AbstractList

成员变量

  • serialVersionUID
  • DEFAULT_CAPACITY:默认初始容量，值为10
  • EMPTY_ELEMENTDATA：为空的列表，共享该数组
  • elementData:数组，用来存储元素。ArrayList的容量就是指这个数组的长度。当第一个元素添加到数组里时，空ArrayList的elementData将会扩充到DEFAULT_CAPACITY
  • size: 现有元素的数量

三个构造方法

1、无构造参数：使用默认值10来初始化数组

/**
    * Constructs an empty list with an initial capacity of ten.
    */
   public ArrayList() {
       super();
       this.elementData = EMPTY_ELEMENTDATA;
   }

2、构造参数为初始容量：先判断initialCapacity是否合法，若合法，将elementData初始化长度为initialCapacity

public ArrayList(int initialCapacity) {
       super();
       if (initialCapacity < 0)
           throw new IllegalArgumentException("Illegal Capacity: "+
                                              initialCapacity);
       this.elementData = new Object[initialCapacity];
   }

3、构造参数为一个集合：取集合c的对象数组，赋给elementData；把数组的长度赋给size。
什么情况下会出现elementData.getClass() != Object[].class？？？？

public ArrayList(Collection<? extends E> c) {
       elementData = c.toArray();
       size = elementData.length;
       // c.toArray might (incorrectly) not return Object[] (see 6260652)
       if (elementData.getClass() != Object[].class)
           elementData = Arrays.copyOf(elementData, size, Object[].class);

add方法

1、 public boolean add(E e):在末尾添加元素，过程如下：modCount（修改次数）加1；ensureCapacityInternal(size + 1)用于判断是否扩充数组，即动态调整大小，其细节在这篇文章里;将元素e添加到List尾部，size（元素个数）加1

public boolean add(E e) {
        ensureCapacityInternal(size + 1);  // Increments modCount!!
        elementData[size++] = e;
        return true;
    }

2、public void add(int index, E element):在指定位置index插入指定元素element：调用rangeCheckForAdd，检测index的合法性；必要时对数组进行扩充；modCount加1;调用System.arraycopy将index位置(及其右边)的元素右移;将index位置的元素改为element

 public void add(int index, E element) {
    rangeCheckForAdd(index);

    ensureCapacityInternal(size + 1);  // Increments modCount!!
    System.arraycopy(elementData, index, elementData, index + 1,
                     size - index);
    elementData[index] = element;
    size++;
}

3、public boolean addAll(Collection<? extends E> c):按照迭代器返回的顺序，将集合c中的元素添加到list中。(为什么一开始不判断c是否为空？)

public boolean addAll(Collection<? extends E> c) {
    Object[] a = c.toArray();
    int numNew = a.length;
    ensureCapacityInternal(size + numNew);  // Increments modCount
    System.arraycopy(a, 0, elementData, size, numNew);
    size += numNew;
    return numNew != 0;
}

4、public boolean addAll(int index, Collection<? extends E> c):将集合c中的元素插入到list的指定位置中去。首先检测index的合法性；modCount加1，必要时扩充数组；将index右边的元素依次右移numNew，最后将c中的元素copy到list中，调整size。

   public boolean addAll(int index, Collection<? extends E> c) {
    rangeCheckForAdd(index);

    Object[] a = c.toArray();
    int numNew = a.length;
    ensureCapacityInternal(size + numNew);  // Increments modCount

    int numMoved = size - index;
    if (numMoved > 0)
        System.arraycopy(elementData, index, elementData, index + numNew,
                         numMoved);

    System.arraycopy(a, 0, elementData, index, numNew);
    size += numNew;
    return numNew != 0;
}

remove方法

1、public E remove(int index):index合法性检测;取到index位置上的值;如果要删除的元素不在最后位置,需要将index之后的元素依次左移,然后将最后位置元素置为null，size减一，返回删除的元素

    public E remove(int index) {
    rangeCheck(index);

    modCount++;
    E oldValue = elementData(index);

    int numMoved = size - index - 1;
    if (numMoved > 0)
        System.arraycopy(elementData, index+1, elementData, index,
                         numMoved);
    elementData[--size] = null; // clear to let GC do its work

    return oldValue;
}

2、private void fastRemove(int index):上一个方法的简化版：不做index合法性检测;不返回删除的元素

   private void fastRemove(int index) {
    modCount++;
    int numMoved = size - index - 1;
    if (numMoved > 0)
        System.arraycopy(elementData, index+1, elementData, index,
                         numMoved);
    elementData[--size] = null; // clear to let GC do its work
}

3、public boolean remove(Object o):删除列表中第一次出现的o
注意点:object为null时的处理

public boolean remove(Object o) {
    if (o == null) {
        for (int index = 0; index < size; index++)
            if (elementData[index] == null) {
                fastRemove(index);
                return true;
            }
    } else {
        for (int index = 0; index < size; index++)
            if (o.equals(elementData[index])) {
                fastRemove(index);
                return true;
            }
    }
    return false;
}

4、protected void removeRange(int fromIndex, int toIndex)：删除fromIndex和toIndex之间的元素(不包括toIndex):用toIndex之后的元素覆盖掉[fromIndex,toIndex)之间的元素，然后将toIndex之后的元素置为null，调整size

 protected void removeRange(int fromIndex, int toIndex) {
    modCount++;
    int numMoved = size - toIndex;
    System.arraycopy(elementData, toIndex, elementData, fromIndex,
                     numMoved);

    // clear to let GC do its work
    int newSize = size - (toIndex-fromIndex);
    for (int i = newSize; i < size; i++) {
        elementData[i] = null;
    }
    size = newSize;
}

5、public boolean removeAll(Collection<?> c):删除list中集合c包含的元素：首先检测c是否为null，然后调用batchRemove

  public boolean removeAll(Collection<?> c) {
    Objects.requireNonNull(c);
    return batchRemove(c, false);
}

6、public boolean retainAll(Collection<?> c)：删除list中不包含在c中的元素

  public boolean retainAll(Collection<?> c) {
    Objects.requireNonNull(c);
    return batchRemove(c, true);
}

7、private boolean batchRemove:这个方法设计的很巧妙，根据complement来确定是要删除还是保留c中的元素。
如果complement为false,表示求与elementData的差集；如果为true，表示求交集。

 private boolean batchRemove(Collection<?> c, boolean complement) {
    final Object[] elementData = this.elementData;
    int r = 0, w = 0;
    boolean modified = false;
    try {
        /**
         *for循环用于筛选想要的元素
         *当complement等于false时，表示从arrayList中删除c中含有的元素。如果c.contains(elementData[r])也等于false，说明c中不包含该元素，这时候要“保存”下这些元素，即非c中的元素，执行elementData[w++]=elementData[r],其实也就是求与elementData的差集
        *当complement等于true时，表示保留arrayList中c中含有的元素，删除其他元素。这时候如果c.contains(elementData[r])等于true，表示c中有该元素，应该保留下来，同样用elementData[w++] = elementData[r]，其他元素忽略掉
         */

        for (; r < size; r++)
            if (c.contains(elementData[r]) == complement)
                elementData[w++] = elementData[r];
    } finally {
        //finally语句用于元素的移动和数组大小调整
        // Preserve behavioral compatibility with AbstractCollection,
        // even if c.contains() throws.
        if (r != size) {
            System.arraycopy(elementData, r,
                             elementData, w,
                             size - r);
            w += size - r;
        }
        if (w != size) {
            // clear to let GC do its work
            for (int i = w; i < size; i++)
                elementData[i] = null;
            modCount += size - w;
            size = w;
            modified = true;
        }
    }
    return modified;
}

Java ArrayList 数组大小动态调整

發表於 2017-11-06   |   分類於 blog

在向ArrayList中添加元素时，都调用了ensureCapacityInternal（）方法，该方法用于调整数组大小。过程如下：
每次调用add操作时，都会对size进行加1操作，然后调用ensureCapacityInternal获得两者较大的那一个，当数组满了的时候，会出现size+1>DEFAULT_CAPACITY的情况，这时候也就需要重新调整数组大小，调用grow进行调整。
grow方法中，数组大小进行调整时，会默认扩大为原来的1.5倍，newCapacity = oldCapacity + (oldCapacity >> 1),然后将newCapacity与size+1进行比较，取两者中较大的一个；作为最终的大小。
如果newCapacity的值超过了VM所允许的最大数组分配个数，就调用hugeCapacity获取最终数组的大小。
最后，调用Arrays.copy,将原先数组中的元素copy到一个更大容量的数组上，然后返回这个新的数组

/**
 *确定数组的容量：
 *如果数组为空，则容量为默认的DEFAULT_CAPACITY--10
 *否则，容量为minCapacity
 */
private void ensureCapacityInternal(int minCapacity) {
        if (elementData == EMPTY_ELEMENTDATA) {
            minCapacity = Math.max(DEFAULT_CAPACITY, minCapacity);
        }

        ensureExplicitCapacity(minCapacity);
    }

/**
 *修改次数加1
 *如果最小容量大于数组当前的容量，调用grow()，进行大小调整
 */

private void ensureExplicitCapacity(int minCapacity) {
        modCount++;

        // overflow-conscious code
        if (minCapacity - elementData.length > 0)
            grow(minCapacity);
    }

/**
     * The maximum size of array to allocate.
     * Some VMs reserve some header words in an array.
     * Attempts to allocate larger arrays may result in
     * OutOfMemoryError: Requested array size exceeds VM limit
     */
    private static final int MAX_ARRAY_SIZE = Integer.MAX_VALUE - 8;

    /**
     * Increases the capacity to ensure that it can hold at least the
     * number of elements specified by the minimum capacity argument.
     *
     * @param minCapacity the desired minimum capacity
     */
    private void grow(int minCapacity) {
        // overflow-conscious code
        int oldCapacity = elementData.length;
        //默认扩展0.5倍
        int newCapacity = oldCapacity + (oldCapacity >> 1);
        if (newCapacity - minCapacity < 0)
            newCapacity = minCapacity;
        if (newCapacity - MAX_ARRAY_SIZE > 0)
            newCapacity = hugeCapacity(minCapacity);
        // minCapacity is usually close to size, so this is a win:
        elementData = Arrays.copyOf(elementData, newCapacity);
    }

    private static int hugeCapacity(int minCapacity) {
        if (minCapacity < 0) // overflow
            throw new OutOfMemoryError();
        return (minCapacity > MAX_ARRAY_SIZE) ?
            Integer.MAX_VALUE :
            MAX_ARRAY_SIZE;
    }

java异常分类

![此处输入图片的描述][1]

  • 如上图，所有的异常类继承自Throwable。
  • 在第二层分为两部分：Error和Exception
  • Error:指java运行时系统内部错误和资源耗尽错误，应用程序不应该抛出这种类型的对象。当出现这种错误时，应该通知给用户，并尽力使程序安全终止。
  • Exception:分为两类：RuntimeException和IOException.划分这两种异常的规则是：由程序错误导致的异常属于RuntimeException;而程序没有错误，由于像I/O错误导致的异常属于其他异常
  • java语言规范将派生于Error类或RuntimeException类的所有异常称为非受查（unchecked）异常；所有其他的异常称为checked异常。
  • RuntimeException通常以下几种情况：
    • 错误的类型转换
    • 数组访问越界
    • 访问null指针
  • 非RuntimeException的异常包括：
    • 访问一个不存在的文件
    • 试图在文件尾部后面读取数据
    • 试图根据给定的字符串查找Class对象，而这个字符串表示的类并不存在

“如果出现RuntimeException异常，那么就一定是你的问题”

###异常的处理

  • 声明受查异常（如果一个方法可能抛出的多个受查异常类型，那么就必须在方法首部列出所有的异常类，每个异常类之间用逗号隔开）
  • 不需要声明java的内部错误，即从Error继承的错误
  • 不需要声明从RuntimeException继承的非受查异常

###如何抛出一个异常

  • 找到一个合适的异常类，并声明该异常
  • 创建这个类的一个对象
  • 将对象抛出

###创建一个异常类

  • 继承Exception或Exception子类
  • 带有两个构造器，一个是默认的构造器；一个是带有详细描述信息的构造器
  • 示例代码如下
class FileFormatException extends Exception{
    public FileFormatException(){}
    public FileFormatException(String gripe){
    super(gripe)
    }
}

###捕获异常
使用try/catch语句块捕捉异常

###finally语句：不管是否有异常被捕获，finally字句中的代码都被执行
示例代码

InputStream in= new FileInputStream(...)
try{
//1
可能抛出异常的代码
//2
}
catch (IOException e){
//3
show error message
//4
}
finally{
//5
in.close
}
//6

可能的代码执行情况：

  • 代码没有抛出异常。执行1、2、5、6
  • 抛出一个在cache子句中捕获的异常，执行1、3、4、5、6
  • 抛出一个异常，但是该异常没有被catch语句捕捉，则执行finally语句，并将异常抛给这个方法的调用者，finally语句后的6并不会执行，这里执行1、5处的语句

####带有return语句的finally：
假设利用return语句从try语句块中退出。在方法返回前，finally子句中等待内容将被执行。如果finally字句中也有一个return语句，这个返回值将会覆盖原始的返回值。示例代码如下

public static int f(int n){
try{
int r = n * n;
return r;
}
finally{
if (n==2)return 0;
}
}
}

如果调用f（2），try语句块中计算结果为4，并执行return语句。然后再执行finally字句，将返回0，覆盖掉原始返回值4.

###带资源的try语句
当在finally语句中进行资源关闭操作时，若close方法抛出异常，这时候就会对close抛出的异常进行处理，代码会非常繁琐。这时可使用带资源的try语句。
示例如下：

try（Scanner in = new Scanner(new FileInputStream("/usr/share/dict/words")),"UTF-8"）
{
while(in.hasNext())
    System.out.println(in.next)
}

这个块在正常退出或存在异常时，会调用in.close()方法，就好像使用了finally块一样。
只需要关闭资源，就要尽可能使用带资源的try语句

DNS查询主机名IP的流程

發表於 2017-10-29   |   分類於 blog

###IP与主机名的历史

  • IP是由32位二进制（IPV4）组成，为了适应人们的习惯，将其转换成四组十进制的数字，如12.34.56.78
  • 由于IP数字这个格式不便于记忆，人们想到了一个方法，在一个文件中记录ip与主机名的对应关系，这个文件就是/etc/hosts，其内容为“IP主机名1主机名2”
  • 当IP数量过多时，导致/etc/hosts文件过大，不利于其他主机的同步。为了解决这个日益严重的问题，伯克利大学发明了域名系统（Domain Name System,DNS）.通过DNS，不需要知道主机的IP，只要知道该主机的名称，就能连上该主机。

###主机名与域名
拿电话号码来解释主机名与域名的关系。
假设有北京、上海各有一个电话号码1234567，那么如何区别这两个号码呢？用区号（区号对应的是域名），假设我想打给上海的1234567，就拨0211234567，同理，打给北京的就拨0101234567。这里021、010就是域名，而1234567是主机名。可见，通过域名来区分相同的主机名。
对于网站，www.google.com,www.seednet.net,www是主机名，那么怎么区分这两个主机名相同的机器呢？答案是通过域名，这里域名是google.com,seednet.net.所以即使两台机器主机名相同，位于不同的域中，也能够被区分出来。

###DNS的阶层架构

如图：
> 最上方是root服务器
> 第二层是顶级域名，如.com,.gov,.edu
> * 第三层是地区级域名，如.cn,.jp

###通过DNS查询主机名IP的流程。

  • 在浏览器中输入http://www.tsinghua.edu.cn时，计算机会根据/etc/resolv.conf这个文件提供的DNS的IP进行连接；
  • DNS服务器在收到用户请求后，先查询本身有没有这个记录，如果没有则向.(root)查询
  • DNS会询问www.tsinghua.edu.cn在哪里，.(root)只记录了.cn信息，则告诉去寻找.cn
  • 同样.cn记录了.edu.cn,.com.cn,.gov.cn,对比后发现，应该去.edu.cn那里寻找，所以告知.edu.cn记录
  • .edu.cn告知应该去tsinghua.edu.cn寻找
  • 在找到tsinghua.edu.cn后发现www这台主机确实是它管理的，则告知www.tsinghua.edu.cn的IP
  • 记录缓存并回报用户
