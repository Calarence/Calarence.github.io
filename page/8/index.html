python编码与解码

發表於 2017-03-13   |   分類於 python

python编码：

计算机是以0-1的方式存储数据的，8比特（8个1）为一个字节，一个字节（11111111）所能表示的最大数字为255，两个字节（11111111 11111111）所能表示的最大数字为65535.
美国人最早研制计算机的时候，将128个字符（字母、数字以及特殊符号）保存到计算机里，用一个字节就够了。这种编码成为ASCII编码。
由于中文用一字节（128）表示不了，UnicodeDecodeError: 'ascii' codec can't decode byte 0xe4 in position 0: ordinal not in range(128) 所以会报这个错误。后来中文有了自己的编码–GB2312。
为了将所有语言统一编码，Unicode应运而生。
Unicode编码用两个字节表示一个字符，这样就会造成一定的浪费：像A完全可以用一个字节表示，第二个字节就浪费掉了。如果表示全英文，用Unicode编码比ASCII编码需要多一倍的存储空间，在存储和传输上就十分不划算。
为了节省空间，UTF-8编码出现了–它是可变长的Unicode编码。UTF-8编码把一个Unicode字符根据不同的数字大小编码成1-6个字节，常用的英文字母被编码成1个字节，汉字通常是3个字节，只有很生僻的字符才会被编码成4-6个字节。
ASCII编码实际上可以被看成是UTF-8编码的一部分，所以，大量只支持ASCII编码的历史遗留软件可以在UTF-8编码下继续工作。

在计算机内存中，统一使用Unicode编码，当需要保存到硬盘或者需要传输的时候，就转换为UTF-8编码.
由于Python的字符串类型是str，在内存中以Unicode表示，一个字符对应若干个字节。如果要在网络上传输，或者保存到磁盘上，就需要把str变为以字节为单位的bytes。
以Unicode表示的str通过encode()方法可以编码为指定的bytes

>>> 'ABC'.encode('ascii')
b'ABC'
>>> '中文'.encode('utf-8')
b'\xe4\xb8\xad\xe6\x96\x87'
>>> '中文'.encode('ascii')
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
UnicodeEncodeError: 'ascii' codec can't encode characters in position 0-1: ordinal not in range(128)

纯英文的str可以用ASCII编码为bytes，内容是一样的，含有中文的str可以用UTF-8编码为bytes。含有中文的str无法用ASCII编码，因为中文编码的范围超过了ASCII编码的范围，Python会报错
如果我们从网络或磁盘上读取了字节流，那么读到的数据就是bytes。要把bytes变为str，就需要用decode()方法
str和bytes互相转换时，需要指定编码。最常用的编码是UTF-8

Redis quick start

發表於 2017-03-10   |   分類於 Redis

大纲

  1. 下载编译Redis
  2. 使用rdis-cli来访问server
  3. 在应用中使用Redis
  4. 更加正确地安装Redis
  5. redis的进一步学习

Redis的安装

推荐使用的Redis安装方式是源码编译安装，所需依赖项为GCC编译器和libc。注意Linux包管理器安装的版本可能不是最新版。
Redis最新稳定版：地址
安装步骤：

wget http://download.redis.io/redis-stable.tar.gz
tar xvzf redis-stable.tar.gz
cd redis-stable
make
sudo make install

启动redis：

redis-server

这个命令在测试环境下可以这样用，在生产环境下，要使用 redis-server /etc/redis.conf，可以用redis源码根目录下的redis.conf作为模板，来写自己的配置文件。

检查Redis是否正常工作

外部程序使用TCP协议和Redis特殊协议同Redis通信。这个特殊协议在Redis client 库中实现，适用于各种语言。为了更简单的使用Redis，Redis提供了命令行工具，来向Redis发送命令。这个工具就是redis-cli。
要检查Redis是否正常工作的第一件事情就是使用redis-cli发送PING命令。

$ redis-cli ping
PONG

redis-cli后的命令以及参数将会发送给localhost，端口为6379的Redis实例。可以通过redis-cli来修改host和port。

另一种运行redis-cli的方式是不带参数：这样将会进入交互模式，可以输入不同的命令来观察回复：

$ redis-cli                                                                
redis 127.0.0.1:6379> ping
PONG
redis 127.0.0.1:6379> set mykey somevalue
OK
redis 127.0.0.1:6379> get mykey
"somevalue"

做到这一步，就能跟Redis通信了。接下来学习Redis的数据类型

centos7下使用nginx+uwsgi部署django项目

發表於 2017-03-03

生产环境搭建：

  1. nginx安装：yum install -y nginx,启动nginx：nginx,访问本机ip地址即可看到nginx欢迎界面
  2. mysql的安装和配置: 链接
  3. virtualenv安装和使用：
    pip install virtualenv
    创建虚拟环境testEnv（在当前目录创建，也可指定其他目录） virtualenv testEnv
    进入虚机环境 source testEnv/bin/activate
    退出虚拟环境 deactivate
  4. 导出项目依赖项：pip freeze > requirements.txt
  5. 虚拟环境中安装依赖项（将第四步导出的txt导入centos中）:pip install -r requirements.txt若某些依赖项安装较慢，可使用源
    pip源设置： vi ~/.pip/pip.conf 内容：
    [global] trusted-host=mirrors.aliyun.com index-url = http://mirrors.aliyun.com/pypi/simple
  6. 测试项目可正常运行 python manage.py runserver
  7. 安装uwsgi：pip install uwsgi,如果报缺少编译器的错误,则安装基础开发包yum groupinstall "Development tools" , yum install -y python-devel
  8. 测试uwsgi可启动django项目:uwsgi --http :8000 --wsgi-file wsgi.py或uwsgi --http :8000 --module yourporject.wsgi注意两个命令执行时的路径，没有错误可正常访问django应用
  9. 重启uwsgi pkill -f uwsgi

  10. nginx配置：新建nginx.conf,添加以下内容：
    `

       # the upstream component nginx needs to connect to
   upstream django {
   # server unix:///path/to/your/mysite/mysite.sock; # for a file socket
   server 127.0.0.1:8000; # for a web port socket (we'll use this first)
   }   
   # configuration of the server
   server {
   # the port your site will be served on
   listen      80;
   # the domain name it will serve for
   server_name 你的ip地址 ; # substitute your machine's IP address or FQDN
   charset     utf-8;
   # max upload size
   client_max_body_size 75M;   # adjust to taste
   # Django media
   location /media  {
   alias 你的目录/yourproject/media;  # 指向django的media目录
   }
   location /static {
   alias 你的目录/yourproject/static; # 指向django的static目录
   }
   # Finally, send all non-media requests to the Django server.
  location / {
  uwsgi_pass  django;
  include     uwsgi_params; # the uwsgi_params file you installed
   }
}

    将该配置文件copy到/etc/nginx/conf.d或sudo ln -s 你的目录/yourproject/conf/nginx/uc_nginx.conf /etc/nginx/conf.d/`

  11. 拉取所有需要的static file 到同一个目录:
    settings.py中添加STATIC_ROOT = os.path.join(BASE_DIR, "static/")
    运行python manage.py collectstatic

  12. 重启nginx:nginx

  13. 通过配置文件启动uwsgi:新建uwsgi.ini 配置文件，内容如下：
    `

    # mysite_uwsgi.ini file
[uwsgi]
# Django-related settings
# the base directory (full path)
chdir           = 你的目录/yourproject
# Django's wsgi file
module          = yourproject.wsgi
# the virtualenv (full path)
# process-related settings
# master
master          = true
# maximum number of worker processes
processes       = 10
# the socket (use the full path to be safe
socket          = 127.0.0.1:8000
# ... with appropriate permissions - may be needed
# chmod-socket    = 664
# clear environment on exit
vacuum          = true
virtualenv = 你的目录/testEnv`

    注意:

    `

chdir： 表示需要操作的目录，也就是项目的目录
module： wsgi文件的路径
processes： 进程数
virtualenv：虚拟环境的目录
`
  14. 启动uwsgi：uwsgi -i 你的目录/uwsgi.ini &
  15. 访问：http://你的ip地址/
  16. 可能遇到的问题：重启nginx或uwsgi时会有端口占用，执行killall -9 uwsgi（或nginx）

centos7下mysql的安装和配置

發表於 2017-02-11   |   分類於 Linux

下载安装mysql-server:

  1. wget http://dev.mysql.com/get/mysql-community-release-el7-5.noarch.rpm
  2. rpm -ivh mysql-community-release-el7-5.noarch.rpm
  3. yum install mysql-community-server

安装完成后重启mysql服务
service mysqld restart

登录mysql(初次登录没有密码)
mysql -u root

设置密码
set password for 'root'@'localhost' =password('password');

再次登录
mysql -u root -p,会提示输入密码，输入密码后，回车即可进入数据库

配置mysql

  1. 编码，编辑/etc/my.cnf，最后添加
    [mysql]
    default-character-set =utf8
    这里的字符编码必须和/usr/share/mysql/charsets/Index.xml中一致

  2. 远程连接
    把在所有数据库的所有表的所有权限赋值给位于所有IP地址的root用户：
    grant all privileges on *.* to root@'%'identified by 'password';
    FLUSH PRIVILEGES

calamari搭建教程

title: calamari搭建教程
标签（空格分隔）： 未分类

  • 架构: 单节点部署ceph，然后在该节点上安装calamari

  • 环境准备:

    • 系统为centos7
    • 关闭防火墙 systemctl stop firewalld systemctl disable firewalld
    • 禁用selinux sed -i 's/SELINUX=.*/SELINUX=disabled/' /etc/selinux/config setenforce 0
    • 修改/etc/hosts文件，增加ip、hostname映射

  • 安装包准备

    • calamari-server rpm包,版本为1.3.3-jewel
    • romana rpm 包,版本为1.2.2-36
    • salt、salt-master、salt-minion rpm包，版本为2015.8.1-1。高版本会出现发现不了集群的问题
    • diamond rpm包

  • 安装：

    • 安装salt rpm -ivh salt-2015.8.1-1.el7.noarch.rpm
    • 安装salt-master rpm -ivh salt-master-2015.8.1-1.el7.noarch.rpm
    • 安装salt-minion rpm -ivh salt-minion-2015.8.1-1.el7.noarch.rpm
    • 安装calamari-server yum install calamari-server-1.3.3-jewel.el7.centos.x86_64.rpm
    • 安装romana rpm -ivh romana-1.2.2-36_gc62bb5b.el7.centos.x86_64.rpm
    • 安装diamond ‘rpm -ivh diamond-3.4.67-0.noarch.rpm’

  • 配置：

    • 将diamond.conf.example重命名为diamond.conf cd /etc/diamond mv diamond.conf.example diamond.conf
    • 将salt-minion的master项配置为salt-master所在机器的主机名 vi /etc/salt/minion master: 主机名 id: node用于标记salt-minion

  • 启动：

    • 启动salt-master: systemctl enable salt-master systemctl restart salt-master
    • calamari初始化 calamarictl initialize,配置用户名密码
    • 访问ip，即可看到登录界面，输入用户名、密码进入后会提示New Calamari Installation,若报500错误，修改cthulhu.log权限cd /var/log/calamari chmod 777 * -R
    • 启动salt-minion：systemctl enable salt-minion systemctl restart salt-minion
    • 启动diamond: systemctl enable diamond systemctl restart diamond
    • salt-key 认证：salt-key -L,会看到salt-minion的标识符，进行认证salt-key -A

  • 验证：

    • salt-minion与master的可达性: salt '*' test.ping,若返回True，则可达
    • 检查salt-master是否能查询到和获取ceph集群 salt '*' ceph.get_heartbeats,若返回集群信息，则可在dashboard上发现集群

  • 日志

    • /var/log/salt/minion
    • /var/log/calamari/calamari.log cthulthu.log
    • diamond

  • 排错

    • 安装完后访问页面报500错误，修改cthulthu.log权限， cd /var/log/calamari chmod 777 * -R
    • 页面提示Cluster Updates Are Stale. The Cluster isn’t updating Calamari. Please contact Administrator，重启salt服务和cthulhu服务看是否出现，如果一直出现可以定期重启，或者进行数据库的清理 数据清理 calamarictl clear --yes-i-am-sure ,然后初始化 calamari-ctl initialize
    • 管理平台看不到监控的硬件数据 发现diamond无法启动 rm -rf /var/lock/subsys/diamond service diamond restart

大型网站架构演化发展历程

發表於 2016-10-18   |   分類於 架构

目的：通过了解网站架构演化历程，明确高可用、高性能、易扩展、可伸缩且安全的实现机制，从而指导网站设计与代码编写

1. 初始阶段的网站架构：应用程序+数据库+文件在一台服务器上，如LAMP。架构如下图所示

2. 应用服务和数据服务分离：应用服务器：需要更强大的cpu，文件服务器需要存储用户上传的文件，需要更大的硬盘；数据库服务器需要磁盘检索和数据缓存，需要更快的硬盘和更大的内存

3. 使用缓存：缓存分为两种，包括缓存在应用服务器上的本地缓存和缓存在专门分布式服务器上的远程缓存。

4. 使用应用服务器集群：通过负载均衡调度服务器改善负载压力

5. 数据库读写分离：配置主从数据库，写操作访问主数据库，通过主从复制将数据同步到从数据库，读操作访问从数据库

6. 使用反向代理和CDN，两者基本原理都是利用缓存，区别是CDN部署在网络提供商的机房，在用户请求网站服务时，可以从距离自己最近的网络提供商厂房获取数据；反向代理部署在网站的机房，当用户请求网站机房时，首先访问反向代理服务器，若反向代理服务器保存用户请求的资源，则将直接返回给用户。

7. 使用分布式文件系统和分布式数据库系统

8. 使用Nosql和搜索引擎

9. 业务拆分：将一个应用拆分成不同的应用，每个应用独立部署维护

10. 分布式服务：将共用业务提取出来独立部署。由可复用的业务连接数据库，提取共用 业务服务，应用系统只需通过分布式服务调用共用业务服务完成具体业务操作

大学这几年看过的闲书

發表於 2016-05-16   |   分類於 读书笔记

写在前面的：
关于读书的意义，看过这么一段：”我读过很多书，但后来大部分都被我忘记了，那读书的意义是什么？” “当我还是个孩子时，我吃过很多的食物，现在已经记不起来吃过什么了。但可以可定的是，它们中的一部分已经长成我的骨头和肉。阅读对思想的改变也是如此！”

关于评分：是按照自己的喜好以及个人感悟对所看书籍的简单评价，没有什么权威性

关于书籍统计工具：自己写的书籍扫描app，通过扫描ISBN码获取书籍信息，用到了Ionic框架，使用这个框架就可以用angularjs开发android、ios app.再一次体会到了nodejs作为全栈式开发语言的强大之处

  1. 《大卫-考坡菲》 个人评分 这是我最喜欢的一本书，虽然作者查尔斯-狄更斯写于1949-1950年（对应到中国的道光帝执政期间），但是仍能引起强烈的共鸣，并没有显得半点out。这是一本半自传的小说，一些章节含有作者自己的影子，将朋友的真诚与阴暗、爱情的幼稚与冲动、婚姻的甜美与琐碎、家人的矛盾与和谐汇聚成一条溪流，在命运的河床上缓缓流淌，最终融入宽容壮美的大海。书中涉及对求学、未来职业、人生伴侣、朋友选择的思考，是一部人生的缩影，虽然有些事情我们还没有经历，但是通过此书可以窥探到将来的事情，遇见时心里也便有了几分底气。
  2. 《你心柔软，却有力量》 个人评分 作家林清玄最新散文精选，书中充满哲学与智慧，一些章节由于我缺乏相应 的经历或者思想不够没有读懂。作者主张用柔软的内心来面对世界，我理解的是这种效果与太极的四两拨千斤有异曲同工之妙。“柔软的心最有力量，唯其柔软，我们才能敏感；唯其柔软，我们才能包容；唯其柔软，我们才能精致；也唯其柔软，我们才能超拔自我，在受伤的时候甚至能包容我们的伤口”
  3. 《朴槿惠自传-绝望锻炼了我》 个人评分 选择这本书的原因是出于对朴槿惠的好奇，好奇的原因是对她经历的一些耳闻以及身上散发的某种气质。这本书作者是朴槿惠，看这本书会发现她的文笔很好，也会对朴槿惠一家在青瓦台的日常生活有所了解。书中的高潮是父亲被刺杀后长达20年的沉寂。
  4. 《普京传》 作者郑文阳 个人评分 普京大帝自然不必多说，每个男人必须了解和学习的领袖。之所以给了三颗星，是因为书中充满了大量的溢美之词，显得过于主观主义，对普京进行了神化，如果语言多一些客观性的描述，则更舒服一点。不过，通过此书可以了解普京从出生到权力巅峰所经历的事件以及如何应对政敌、寡头、车臣之战，包括所遭遇到12次暗杀。男士必读，力量之作。
  5. 《白夜行》 未读完。。。东野圭吾作品，我读过的第一本日本作家，但是读了几章就停下了，书中大量人物角色以及他们的关系实在搞不清，加之同学说这本小说比较压抑就放弃了。。。
  6. 《第一日》 个人评分 马克-李维的科幻、探险、爱情小说，一个女考古学家和一个天体物理学家围绕各自的理想和追求展开的偶有交集的最后到底交集在一起的爱情故事，结合物理、化学、生物、地理、天文、考古等科学事实和基础，低调奢华有内涵。
  7. 《第一夜》第一日的续集。

  51. 《看见》个人评分 《看见》中，我没有刻意选择标志性事件，也没有描绘历史的雄心，在大量的新闻报道里，我只选择了留给我强烈生命印象的人，因为工作原因，我恰好与这些人相遇。他们是流淌的，从我心腹深处的石坝上漫溢出来，坚硬的成见和模式被一遍遍冲刷，摇摇欲坠，土崩瓦解。这种摇晃是危险的，但思想的本质就是不安。
    我试着尽可能诚实地写下这不断犯错、不断推翻、不断疑问、不断重建的事实和因果，一个国家由人构成，一个人也由无数他人构成，你想如何报道一个国家，就要如何报道自已。——柴静
  52. 《羊道》个人评分 李娟与哈萨克牧民扎克拜妈妈一家共同生活、历经寒暑跋涉后，在几年时间内陆续写下的文字.在李娟笔下，这支也许是世界上仅存的、真正意义上的游牧民族的生存景观得以呈现。这是一种与大自然生死相依，充满了艰辛、苦难而又自有其尊严与乐趣的古老生活。李娟用自己的方式讲述着这些世界角落的人和事：“所有的文字都在强调他们的与众不同。而我，我更感动于他们与世人相同的那部分。那些相同的欢乐，相同的忧虑与相同的希望。”
  53. 《我的灵魂永不下跪-临刑前的最后一课》 尊严对于将死之人意味着什么？或许，从这本书中能找到答案。
  54. 《肖申克的救赎》看小说跟电影的感觉还是不一样的，推荐。

关于整洁的代码

發表於 2016-04-04   |   分類於 读书笔记

人们常说“字如其人”，认为一个人写字好看与其长相成正比，字写得好看，人就长的越漂亮。对此，我一直持怀疑态度。因为自己字的并不好看，但长的还。。。但对于代码整洁与否，我觉得能反应出一个程序猿的专业素养：包括专业知识的掌握程度以及是对优雅的追求。整洁的代码如同一个人穿了一件干净、得体、简单大方的衣服，看起来让人觉得舒服，而混乱的代码只能让人觉得难受、痛苦

那么对于“整洁的代码”，又有什么样的定义呢？《代码整洁之道》在第一章采访了一些非常知名且经验丰富的程序员


Dave具体到了整洁代码的细节，如良好的命名，只做一件事，尽量少的的API..而这些细节是我们很容易注意到的，并没有高大上之处。

然而写出整洁代码并非一日之功，需要遵循大量的小技巧，贯彻刻苦习得的“整洁感”。但是只要我们有了整洁代码的意识，写代码时注意思考与重构，相信写出优雅代码只不过是时间问题
