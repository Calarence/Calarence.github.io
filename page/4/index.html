java内存分配与回收策略

發表於 2018-01-05   |   分類於 blog

java内存分配与回收策略

  1. 对象优先在Eden分配：大多数情况下，对象在新生代Eden区中分配。当Eden区没有足够的空间时，虚拟机将发起一次Minor GC。如果GC时发现已有对象无法放入Survivor空间，则通过分配担保机制提前转移到老年代中去。
  2. 大对象直接进入老年代。大对象是指需要大量连续内存空间的java对象，最典型的大对象如很长的字符串以及数组。虚拟机提供了一个-XX：PretenureSizeThresold参数，令大于该值的对象直接才老年代分配
  3. 长期存活的对象进入老年代：虚拟机给每个对象定义了一个对象年龄计数器。通过该计数器来判断是否需要进入老年代。如果对象在Eden出生并且经过第一次Minor GC仍然能够存活下来，并且能被survivor容纳，将被移动到survivor空间中，并且年龄设为1。对象在Survivor区中每熬过一次Minor GC，年龄就增长一岁。当它的年龄增加到一定程度，默认为15岁，就会被移动到老年代中。对象晋升老年代的年龄阀值，可以通过参数-XX：MaxTenuringThresold来设置。
  4. 动态对象年龄判定：为了能更好地适应不同程序的内存状况，虚拟机并不是永远地要求对象的年龄必须达到了MaxTenuringThresold才能晋升老年代。如果在Survivor空间中，相同年龄所有对象的大小的总和大于Survivor空间的一半，年龄大于或等于该年龄的对象就可以直接进入到老年代中，无须等到MaxTenuringThresold所要求的年龄。
  5. 空间分配担保：在Minor GC之前，虚拟机会检查老年代最大可用的连续空间是否大于新生代所有对象总空间，如果这个条件成立，那么Minor GC可以确保是安全的。如果不成立，则查看HandlePromotionFailure设置是否允许担保失败。如果允许，那么会继续检查老年代最大可用连续空间是否大于历次晋升到老年代对象的平均大小，如果大于，则会尝试进行Minor GC；如果小于或者不允许担保失败，则进行一次Full GC。大部分情况下会打开HandlePromotionFailure，以避免Full GC过于频繁。

java垃圾收集

發表於 2018-01-05   |   分類於 blog

java GC

GC需要考虑的三个问题：

  1. 哪些内存需要回收
  2. 何时回收
  3. 如何回收

对这三个问题的解决，也是本文的叙述顺序

1、哪些内存需要回收–可达分析算法来判断：
由于引用计数法无法解决循环引用的问题，所以在java中没有采用该方法，而是采用了一种叫做“可达分析”的算法。
这个算法的基本思路是通过一些列称为“GC Roots”的对象作为起始点，从这些起始点开始向下搜索，搜索走过的路径称为引用链。当一个对象到GC Roots没有任何引用链，也就是从GC Roots到这个对象不可达，则证明此对象时不可用的，由此判定这些对象是可回收的对象。
可作为GC Roots的对象包括下面几种：

  • 虚拟机栈中（本地变量表）引用的对象
  • 方法区中类静态属性引用的对象
  • 方法区中常量引用的对象
  • 本地方法栈中引用的对象

2、何时回收–安全点、安全区域才进行GC：
程序执行时并非在所有的地方都能停下来开始GC，只有到达安全点时才能暂停。
3、如何回收：回收算法（理论）+垃圾收集器（实现）：
回收算法：
> 标记清除：
算法分为两个阶段：标记（可达分析算法）和清除。它有两个不足：一个是效率问题，标记和清除这两个过程效率都不高；第二个是空间问题，会产生空间碎片。空间碎片太多可能会导致以后分配大对象时，无法找到足够内存而不得不提前触发另一次垃圾收集动作。
> 复制：将内存一分为二，每次只使用其中一块。当一块用完后，就将上面还存活的对象复制到另一块上，然后将使用过的内存一次性的回收。
优点：实现简单，运行高效
缺点：内存缩小为原来的一半，代价高昂
用于回收新生代对象
> 标记整理：标记阶段与‘标记-清除’方法一样，但后续步骤不是直接对可回收对象进行清理，而是让所有存活的对象都移动到一端，然后直接清理掉端边界以外的内存。
> 分代收集算法：将java堆分为新生代和老年代，新生代采用复制算法，而老年代采用标记-清除或标记整理算法。

垃圾收集器:

  • Serial收集器：
    最基本的、发展历史最悠久的新生代收集器；单线程，工作时会暂停其他所有的工作线程；虚拟机运行在client模式下的默认新生代收集器
  • ParNew收集器：
    Serial收集器的多线程版本；运行在server模式下虚拟机首选的新生代收集器；除Serial外，唯一的能与CMS收集器配合工作的收集器
  • Parallel Scavenge收集器：
    新生代收集器；并行多线程；与其他收集器关注用户线程停顿时间不一样，Parallel收集器关注的是吞吐量，吞吐量优先的收集器；Parallel Scavenage收集器提供了两个参数用于精确控制吞吐量，分别是控制最大停顿时间-XX:MaxGCPauseMillis参数以及设置吞吐量大小的-XX:GCTimeRatio；此外，还有一个参数-XX:+UseAdaptiveSizePolicy,这是一个开关参数，当这个参数打开后，就不需要手动指定新生代的大小（-Xmn），Eden与Survivor的比例(-XSS:SurvvivorRatio)、晋升老年代对象大小（-XX：PretenureSizeThreshold）等细节参数了，虚拟机会根据当前系统的运行情况收集性能监控信息，动态调整这些参数以提供最合适的停顿时间或者最大吞吐量，这种调节方式称为GC自适应的调节策略。
  • Serial Old收集器：Serial收集器的老年代版本；使用标记-整理算法；clint模式下的虚拟机使用
  • Parallel Old收集器：Parallel Scavenage收集器的老年代版本。
  • CMS（Concurrent Mark Sweep）收集器:并发标记、低停顿，以获取最短停顿时间为目标，采用标记-清除算法；服务端；老年代。
    运行过程：
  • 初始标记：仅标记GC Roots能直接关联到的对象，暂停用户线程
  • 并发标记：从GC Roots开始对堆中对象进行可达性分析，找出存活的对象，此过程时间较长，但可与用户线程并发执行
  • 重新标记：修正并发标记期间因用户程序继续运作而导致标记产生变动的那一部分对象的标记记录。
  • 并发清除：与用户线程并发执行
    三个缺点：
  • CMS收集器对CPU资源非常敏感
  • CMS收集器无法处理浮动的垃圾
  • 基于标记-清除，会产生空间碎片
  • G1收集器：面向服务端；并发与并行；分代收集；空间整合（使用标记-整理）；可预测的停顿，除了追求低停顿外，还能建立可预测的停顿时间模型，能让使用者指定在一个长度为M毫秒的时间片段内，消耗在垃圾收集上的时间不超过N毫秒；
    使用G1收集器时，Java堆的内存布局就与其他收集器有很大差别，它将整个java堆划分为多个大小相等的独立区域（Region）,虽然还保存着新生代与老年代，但他们不再是物理隔离的了，他们都是一部分Region的集合。
G1收集器之所以能建立可预测的停顿时间模型，是因为它可以跟踪各个Region里面的垃圾堆积的价值大小（回收所获得的的空间大小以及回收所需时间的经验值），在后台维护一个优先列表，每次根据允许的收集时间，优先回收价值最大的Region
过程：前三个过程与CMS收集器过程基本类似
初始标记
并发标记
最终标记
筛选回收：对各个Region的回收价值和成本进行排序，根据用户制定的GC停顿时间来制定回收计划。

java内存布局

發表於 2018-01-05   |   分類於 blog

#java内存大概分为以下几个区域：

  • 程序计数器：用于记录线程所要执行的字节码指令的地址；程序计数器是一块较小的内存，线程私有；当执行的是java方法是，存储的是字节码指令的地址；如果执行的是native方法，则为空（undefined），该内存区域是唯一一个不会出现OutOfMemoryError的区域。
  • 虚拟机栈：线程私有；每个方法在执行的时候，都会创建一个栈帧，用于存储局部变量表（基本数据类型、对象引用）、操作数栈、动态链接、方法出口等信息。一个方法从调用到执行完成的过程，对应着一个栈帧在虚拟机栈从入栈到出栈的过程。
  • 本地方法栈：虚拟机栈用于执行java方法，而本地方法栈用于执行native方法。
  • 方法区：用于存储加载的类的信息（版本、方法、接口、字段）、常量、静态变量、即时编译器编译后的代码；线程共享。
  • 运行时常量池：是方法区的一部分，用于存储编译期编译生成的各种字面量和符号引用
  • 堆：用于存储对象实例以及数组。线程共享

  1. java堆溢出
    原理：java堆用于存储对象实例，只要不断地创建对象，并保证GC Roots到对象之间有可达路径来避免垃圾回收机制来清除这些对象，那么在对象数量到达最大堆容量限制后就会产生内存溢出异常。

    import java.util.List;

    }

     List<OOMObject> list =new ArrayList<OOMObject>();
    while(true){
        list.add(new OOMObject());
    }
     }
   } }

运行方式：java -Xms20m -Xmx20m -XX:+HeapDumpOnOutOfMemoryError

    对常量池的测试
    原理：String.intern()是一个nanative方法，如果字符串常量池中已经包含一个等于此字符串对象的字符串，则返回代表池中这个字符串的Strig对象；否则，将String对象包含的字符串添加到常量池中，并且返回此Strig对象的引用。
    JDK1.6下：
    示例代码：

    import java.util.ArrayList;
    import java.util.List;
    public class RuntimeConstantPoolOOM {
    public static void main(String[] args) {

    List<String> list = new ArrayList<String>();
int i = 0;
while (true) {
    list.add(String.valueOf(i++).intern());
}

    }
    }
    运行方式：java -XX:PermSize=10M -XX:MaxPermSize=10M
    输出结果：Exception in thread “main” java.lang.OutOfMemoryError：PermGen space

    at java.lang.String.intern(Native Method)

注意：在JDK1.7下，该程序将一直进行下去。原因是intern方法在JDK1.7中发生了改变。intern方法不再复制实例，只是在常量池中记录首次出现的实例引用。

Java基础

Java基础回顾

  • 大小写敏感
  • 4个访问修饰符及范围
  • 类的命名规范：类名是以大写字母开头。驼峰命名
  • 源代码的文件名必须与public类的名字相同，并以.java作为扩展名
  • 使用javac xxx.java编译程序，使用java xxx来运行程序。xxx这里值类名
  • 注释：有三种
    • 行注释: //something
    • 段注释： / something /
    • 可自动生成文档的注释： /* something /
  • 基本数据类型：共8种：4种整型，2种浮点型，1种字符型，1种布尔型
    • 整型：范围从小到大依次为：byte、short、int、long，所需要的存储空间依次为1字节、2字节、4字节、8字节。java中所有数值类型所占据的字节数量与平台无关
    • 浮点型：float、double，double的精度是float类型的两倍，也称为双精度。所需存储空间以此为4字节、8字节
    • 字符类型：char。编码格式的相关内容
    • 布尔类型：boolean。整数值和布尔值之间不能进行相互转换
  • 引用类型：数组、类、接口
  • 变量：变量的声明以及命名规范；变量的初始化；在java中，变量的声明尽可能地靠近变量第一次使用的地方，这是一种良好的程序编写风格
  • 常量：用关键字final修饰常量，表示该变量智能被赋值一次。一旦被赋值之后，就不能被修改了。常量名使用全大写。如果希望某个常量可以在一个类中的多个方法中使用，通常将这些变量称为类常量，使用static final来设置一个类常量。类常量的定义位于main方法外部
  • 运算符：+，-，*，/。当参与/运算的两个操作数都是整数时，表示整数除法，否则表示浮点除法。整数被0整除会产生一个异常，而浮点数被0整除将会得到无穷大或NaN结果
  • 数值类型之间的转换：实心箭头表示无信息丢失的转换；虚箭头表示可能有精度损失的转换。

    n+f,n是整数，f是浮点数，需要将两个操作数转换为同一种类型，再进行运算。
    • 如果两个操作数中有一个是double类型，另一个操作数就会转换为double类型
    • 如果一个操作数是float类型，另一个操作数机就会转换为float类型
    • 如果一个操作数是long类型，另一个操作数就会转换为long类型
    • 否则，两个操作数都将转换为int类型。
  • 强制类型转换
  • 流程控制
    • 块作用域：块决定了变量的作用域；不能在嵌套的两个块中声明同名的变量
    • if else
    • while、do…while
    • for循环：
    • switch case:case标签可以是char/byte/short/int的常量表达式；或枚举常量；从java SE7开始，case标签还可以是字符串字面量
    • 中断控制流程语句：break，continue
  • 字符串
  • 大数值：BigInteger和BigDecimal：这两个类可以处理包含任意长度数字序列的数值
  • 数组
    • 数组拷贝：使用Arrays类的copyOf方法会将一个数组的所有值copy到一个新的数组中去，这时候对新数组的修改将不会影响到原先数组。

Java接口

發表於 2017-11-27   |   分類於 blog

接口基本特性

  • 接口中所有的方法默认为public
  • 接口中可以定义常量，自动设为public static final
  • 接口中不能含有实例域，因为接口没有实例
  • java8之前，接口中不能定义简单方法
  • 在实现接口时，必须把方法声明为public，否则，编译器会认为这个方法的访问属性是包可见性，即类的默认访问属性，之后编译器就会给出试图提供更严格的访问权限的警告信息。
  • 不能使用new来构造接口的对象，但可以声明接口的变量，接口变量必须引用实现了接口的类对象
  • 可以使用instanceof来检查一个对象是否实现了某个特定的接口
  • 每个类只能继承一个超类，但可以实现多个接口
  • java8中，允许在接口中增加静态方法。之前都是将静态方法放在伴随类中，出现接口和使用工具类，如Collection/Collections,现在可以在接口中定义静态方法了
  • 默认方法：可以为接口方法提供一个默认实现，必须用default修饰符 标记这样一个方法；默认方法可以调用任何其他方法

接口与抽象类

  • 在java中，每个类只能继承一个类，即不支持多重继承，其主要原因是多重继承会让语言本身变得非常复杂（如同C++），效率也会降低（如同Eiffel）
  • 接口可以提供多重继承的大多数好处，同时还能避免多重继承的复杂性和低效性

解决默认方法冲突

如果现在一个接口中将一个方法定义为默认方法，然后又在超类或另一个接口中定义了同样的方法，那么子类会如何出现这种二义性的冲突。

  • 超类优先。如果超类提供了一个具体方法，同名而且有相同参数类型的默认方法会被忽略
  • 接口冲突：如果一个接口提供了一个默认方法，另一个接口提供了一个同名而且参数类型（不论是否是默认参数）相同的方法，必须覆盖这个方法来解决冲突

Java对象与类

Java对象与类

类

对象：三个主要特性：行为、状态、标识

类之间的关系：依赖（use a）、聚合（has a）、继承（is a）.应该将相互依赖的类减至最少，即让类之间的耦合度最小

隐式参数与显式参数：隐式参数是出现在方法名前面的某个类的对象；显式参数是位于方法名后面括号中的值

方法参数：Java采用按值调用。

方法得到的是参数值的一个拷贝，方法不能修改传递给它的任何参数变量的内容。
举例如下：

public static void tripleValue(double x)
    {
     x = 3 * ;

    }
//调用该方法
double percent = 10;
tripleValue(percent);

执行完后，发现percent的值还是为10，具体执行过程如下:

  • x被初始化为percent的一个copy，也就是10
  • x被乘以3后等于30，但percent仍然是10
  • 该方法结束后，x不被使用
  • percent仍为10

方法参数共有两种类型：基本数据类型；对象引用。在上面例子中，已经看到了一个方法不能改变一个基本数据类型的参数，那么对象引用类型的参数呢？

public static void tripleSalary(Employee x)
{
 x.raiseSalary(200);
} 
当调用：
harry = new Employee();
tripleSalary(harry);

具体过程如下:

  • x被初始化为harry值的一个copy，这里是同一个对象的引用。
  • raiseSalary应用于这个对象的引用。x和harry同时引用的那个Employee对象的salary改变
  • 方法结束后，参数x不被使用，对象变量harry继续引用salary改变的对象

参数得到是对象引用的copy，对象引用以及他的copy同时引用同一个对象。

java中的引用不是按引用调用，反例如下:

public staic void swap(Employee x,Employee y)
 {
   Employee temp = x;
   x =  y;
   y = temp；
 }

如果java对对象采用的是按引用调用，那么上面的方法就能实现交换数据的效果，但是实际上并没有改变存储在变量a和b中的的对象引用。swap方法的参数x和y被初始化为两个对象应用的copy，这个方法交换的是两个copy

总结：

  • 一个方法不能修改一个基本数据类型的参数
  • 一个方法可以改变一个对象参数的状态
  • 一个方法不能让对象参数引用一个新的对象

重载:如果有多个方法由相同的名字、不同的参数，便产生了重载。方法名以及参数类型，叫作方法的签名。返回类型不是方法签名的一部分。也就是说不能有两个名字相同，参数类型也相同，但是返回不同类型值的方法。

静态方法

静态方法是一种不能向对象实施操作的方法。静态方法不能访问成员变量，因为它不能操作对象。
在下面两种情况使用静态方法：

  • 一个方法不需要访问对象状态，其所需参数都是通过显式参数提供
  • 一个方法只需要访问类的静态域

初始化块

初始化数据的方法：

  • 在构造器中赋值
  • 在声明中赋值
  • 初始化块：首先运行初始化块，然后才运行构造器的主体部分。

调用构造器的处理步骤;

  • 所有数据被初始化默认值（0，null或false）
  • 按照在类声明中出现的次序，依次执行所有域初始化语句和初始化块
  • 如果构造器中第一行调用了第二个构造器，则执行第二个构造器主体
  • 执行这个构造器主体

对静态变量的初始化：静态的初始化块，在类第一次加载的时候，会进行静态域的初始化，所有静态初始化语句以及静态初始化块都依照类定义的顺序执行。

类设计技巧

  • 要保证成员变量私有
  • 要对成员变量初始化
  • 不要在类中使用过多的基本类型，可用其他类代替多个相关的基本类型的使用
  • 将职责过多的类进行分解
  • 不是所有的域都需要独立的get和set方法
  • 类名和方法名要能体现他们的职责
  • 优先使用不可的类

其他

  • 一个对象变量并没有实际包含一个对象，而仅仅引用一个对象
  • 在java中，任何对象变量的值都是对存储在另外一个地方的一个对象的引用。new 操作符的返回值也是一个引用
  • 在一个源文件中，只能有一个public类，但可以有任意数目的非公有类

###为什么需要Django执行SQL
使用Django自带的ORM机制做查询有时会满足不了我们的需求， 例如某些查询没有对应的model；或者只想执行UPDATE、INSERT、或DELETE语句。在这些情况下，我们需要直接访问数据库，并想绕过model层。那么，我们应该如何做呢？
django.db.connection表示当前默认数据库的连接。如果想使用该连接，调用connection.cursor()来获得一个cursor对象；接着，调用cursor.execute(sql, [params])来执行SQL语句；然后使用cursor.fetchone()或cursor.fetchall()来返回查询结果。
示例如下：
from django.db import connection

def my_custom_sql(self):
    with connection.cursor() as cursor:
        cursor.execute("UPDATE bar SET foo = 1 WHERE baz = %s", [self.baz])
        cursor.execute("SELECT foo FROM bar WHERE baz = %s", [self.baz])
        row = cursor.fetchone()
    return row

注意：如果想在查询语句中包含百分号，必须写两个，示例如下

cursor.execute("SELECT foo FROM bar WHERE baz = '30%'")
cursor.execute("SELECT foo FROM bar WHERE baz = '30%%' AND id = %s", [self.id])

如果正在使用多个数据库，可以使用django.db.connections来获取指定数据库的连接（和cursor）。 django.db.connections是一个类似字典的对象，可以使用别名来获取指定的连接。

from django.db import connections
cursor = connections['my_db_alias'].cursor()
# Your code here...

默认情况下，Python DB API返回的结果中不包含字段名，这表示最终结果是值的list，而不是dict。可以使用以下代码返回一个dict。

def dictfetchall(cursor):
    "Return all rows from a cursor as a dict"
    columns = [col[0] for col in cursor.description]
    return [
        dict(zip(columns, row))
        for row in cursor.fetchall()
    ]

另一个选择是使用python标准库 collections.namedtuple()。namedtuple是类似tuple的对象，支持属性查询来访问字段；同时也支持索引和迭代。结果是不可变的，并且可通过字段名或切片访问，这两个特性可能会比较有用。

from collections import namedtuple

def namedtuplefetchall(cursor):
"Return all rows from a cursor as a namedtuple"
    desc = cursor.description
    nt_result = namedtuple('Result', [col[0] for col in desc])
    return [nt_result(*row) for row in cursor.fetchall()]

以上三种情况的差别如下所示：

>>> cursor.execute("SELECT id, parent_id FROM test LIMIT 2");
>>> cursor.fetchall()
((54360982, None), (54360880, None))

>>> cursor.execute("SELECT id, parent_id FROM test LIMIT 2");
>>> dictfetchall(cursor)
{'parent_id': None, 'id': 54360982}, {'parent_id': None, 'id': 54360880}]

>>> cursor.execute("SELECT id, parent_id FROM test LIMIT 2");
>>> results = namedtuplefetchall(cursor)
>>> results
[Result(id=54360982, parent_id=None), Result(id=54360880, parent_id=None)]
>>> results[0].id
54360982
>>> results[0][0]
54360982

  • 每个迭代器检查自己改写操作的计数值是否与集合的改写操作计数值一致。如果不一致，则抛出ConcurrentModificationException
final void checkForComodification() {
       if (modCount != expectedModCount)
           throw new ConcurrentModificationException();
   }
  • 返回cursor指向的元素，然后cursor加1.这个方法会迷惑人，从字面意义上觉得是返回下一个元素，但事实却是返回当前元素，然后将cursor指向下一个元素。所以，next可以理解为将cursor指向next。
public E next() {
        checkForComodification();
        int i = cursor;
        if (i >= size)
            throw new NoSuchElementException();
        Object[] elementData = ArrayList.this.elementData;
        if (i >= elementData.length)
            throw new ConcurrentModificationException();
        cursor = i + 1;
        return (E) elementData[lastRet = i];
    }
  • remove方法：在调用remove方法之前，如果没有调用next方法，将会抛出IllegalStateException,原因是lastRet初始值为-1，小于0，抛出IllegalStateException;而调用next方法后，将lastRet赋值为cursor，即当前访问的元素，这样lastRet大于或等于0，不会抛出异常。然后调用ArrayList的remove方法删除元素；又将lastRest复位为-1，所以下次remove时，需要再次调用next
public void remove() {
        if (lastRet < 0)
            throw new IllegalStateException();
        checkForComodification();

        try {
            ArrayList.this.remove(lastRet);
            cursor = lastRet;
            lastRet = -1;
            expectedModCount = modCount;
        } catch (IndexOutOfBoundsException ex) {
            throw new ConcurrentModificationException();
        }
    }

1、 public boolean add(E e):在末尾添加元素，过程如下：modCount（修改次数）加1；ensureCapacityInternal(size + 1)用于判断是否扩充数组，即动态调整大小，其细节在这篇文章里;将元素e添加到List尾部，size（元素个数）加1

    ensureCapacityInternal(size + 1);  // Increments modCount!!
    System.arraycopy(elementData, index, elementData, index + 1,
                     size - index);

}

7、private boolean batchRemove:这个方法设计的很巧妙，根据complement来确定是要删除还是保留c中的元素。
如果complement为false,表示求与elementData的差集；如果为true，表示求交集。

 private boolean batchRemove(Collection<?> c, boolean complement) {
    final Object[] elementData = this.elementData;
    int r = 0, w = 0;
    boolean modified = false;
    try {
        /**
         *for循环用于筛选想要的元素
         *当complement等于false时，表示从arrayList中删除c中含有的元素。如果c.contains(elementData[r])也等于false，说明c中不包含该元素，这时候要“保存”下这些元素，即非c中的元素，执行elementData[w++]=elementData[r],其实也就是求与elementData的差集
        *当complement等于true时，表示保留arrayList中c中含有的元素，删除其他元素。这时候如果c.contains(elementData[r])等于true，表示c中有该元素，应该保留下来，同样用elementData[w++] = elementData[r]，其他元素忽略掉
         */

        for (; r < size; r++)
            if (c.contains(elementData[r]) == complement)
                elementData[w++] = elementData[r];
    } finally {
        //finally语句用于元素的移动和数组大小调整
        // Preserve behavioral compatibility with AbstractCollection,
        // even if c.contains() throws.
        if (r != size) {
            System.arraycopy(elementData, r,
                             elementData, w,
                             size - r);
            w += size - r;
        }
        if (w != size) {
            // clear to let GC do its work
            for (int i = w; i < size; i++)
                elementData[i] = null;
            modCount += size - w;
            size = w;
            modified = true;
        }
    }
    return modified;
}

1、缩减长度：减少ArrayList的存储。如果数组当前长度小于容量（也就是说还有剩余容量），就把剩余容量删除掉

public void trimToSize() {
        modCount++;
        if (size < elementData.length) {
            elementData = Arrays.copyOf(elementData, size);
        }
    }

2、clear方法：修改次数加1；遍历数组，依次将各个元素置为null，然后将size置为0

public void clear() {
    modCount++;

    // clear to let GC do its work
    for (int i = 0; i < size; i++)
        elementData[i] = null;

    size = 0;
}

3、set方法：public E set(int index, E element)
首先检测index合法性；若合法，取出原先index位置上的值；将index上值进行更新；返回原先的值

public E set(int index, E element) {
    rangeCheck(index);
    E oldValue = elementData(index);
    elementData[index] = element;
    return oldValue;
}
