scaling and architecture
本章主要讲了实现系统伸缩性与可扩展性的几个方法：多线程与多进程，事件驱动架构、面向服务的架构，其中在讲面向服务的架构时提到了openstack使用的消息协议机制

  1. 多线程： It’sthe ability torun code on separate processors inside a single Python process. This means that different parts of your code will be run in parallel.多线程可能是提高应用伸缩性和并行化处理的一种方式，但是会受到GLI（The GIL is the Python Global InterpreterLock,alock that must be acquired each time CPython needs to execute byte-code）的限制
  2. 多线程与多进程:As explained earlier, multi-threading is not a good scalability solution because of the GIL. A better solution is the multiprocessing package that is provided with Python.
    It provides the same kind of interface that you would have using the multithread-ing module,except that it starts new processes(viafork(?)) rather than new system
    threads.多进程与多线程相比，在cpu利用方面有很大节省
    总结：the multithreading module is not only able to efficiently spread a work-loads over several local processors,but can also do so over a network,through its multithreading.managers objects. It also provides bi-directional communica-tion transports so your processes can exchange information with each other.Each time you think that you can parallelize some work for a certain amount of time, it’s much better to rely on multi-processing and to fork your jobs, in order to spread the workload among several CPU cores
  3. 异步与事件驱动架构:Event-driven programming is a good solution to organize program flow in a way which listens for various events at once,without using a multi-threaded approach.事件驱动背后的机制是：建立一个event loop。程序会阻塞直到一个事件到达。当等待输入或完成输出时，程序处于busy状态。最基本打事件是”I have data ready to be read” or “I can now write data without blocking”.
  4. 面向服务架构：SOA is the architecture type used by OpenStack in all its components. Components use HTTP REST to communicate with external clients(end-users) and an abstracted RPC mechanism that can support several wire protocols,the mostcommonly used one being AMQP

Data structures
python提供了许多数据结构，来使我们的代码更加优雅和简单。有时候，我们可能更想以自己的方式来实现数据结构，但是this is invariably a vain, useless, doomed idea.学习使用Python提供的数据结构是更好的选择。
作者用dict举了一个例子

def get_fruits(basket, fruit):
# A variation is to use "if fruit in basket:"
try:
    return basket[fruit]
except KeyError:
    return set()

但是使用dict提供的get方法显得更加简单

def get_fruits(basket, fruit):
    return basket.get(fruit, set())

最后作者给出了建议：By the way, the collections module offers a few useful data structures that can solve other kinds of problems, such as OrderedDict or Counter.It’s really important to look for the right data structure in Python, as the correct choice will save you time, and lessen code maintenance.

ordered list and bisect
在对大列表进行操作时，使用有序列表与无序列表相比，有许多优点，如有序列表的存取时间为O（log n).
python提供了bisect模块，该模块包含了bisection算法。bisect函数能够让我们获取新元素插入位置的同时保持列表有序

可以通过以下代码创建一个有序列表

import bisect
class SortedList(list):
    def __init__(self, iterable):
        super(SortedList, self).__init__(sorted(iterable))
    def insort(self, item):
        bisect.insort(self, item)
    def index(self, value, start=None, stop=None):
        place = bisect.bisect_left(self[start:stop], value)
        if start:
            place += start
           end = stop or len(self)
       if place < end and self[place] == value:
           return place
       raise ValueError("%s is not in list" % value)

注意：不要使用appen或extend,否则列表永远不会有序

Namedtuple and slots
有时候我们会创建仅含有几个属性的简单对象，一个简单的实现方式如下：

class Point(object):
    def __init__(self, x, y):
        self.x = x
        self.y = y

这样写的优点是可以随便增加想要的属性；缺点是使用字典来存储这些属性对内存来说是非常昂贵的，因为需要存储key、value以及reference

优化方法：使用slots。Classes in Python can define a slots attribute that will list the only attributes allowed for instances of this class. 它的优势在于不是分配一个整个的字典对象来存储所有的对象属性，是用list来存储。

slots原理：Python converts the content of slots into a tuple, then a list that it builds and sorts, before converting it back into a tuple to use and store it in the class. This way, Python can retrieve the values quickly,without having to allocate and use an entire dictionary

slots好处：通过使用slots可以节约一半的内存，这意味着当处理大量的简单对象时，slots属性是一个高效有效的选择

Memoization
Memoization是通过缓存函数执行结果来加速函数调用的一种机制。只有函数为纯函数(表示该函数没有副作用或输出，不依赖任何全局东西)时,结果才可以被缓存

sin函数示例

>>> import math
>>> _SIN_MEMOIZED_VALUES = {}
>>> def memoized_sin(x):
... if x not in _SIN_MEMOIZED_VALUES:
... _SIN_MEMOIZED_VALUES[x] = math.sin(x)
... return _SIN_MEMOIZED_VALUES[x]
>>> memoized_sin(1)
0.8414709848078965
>>> _SIN_MEMOIZED_VALUES
{1: 0.8414709848078965}
>>> memoized_sin(2)
0.9092974268256817
>>> memoized_sin(2)
0.9092974268256817
>>> _SIN_MEMOIZED_VALUES
{1: 0.8414709848078965, 2: 0.9092974268256817}
>>> memoized_sin(1)
0.8414709848078965
>>> _SIN_MEMOIZED_VALUES
{1: 0.8414709848078965, 2: 0.9092974268256817}

memoized_sin在第一次调用时，参数没有保存在_SIN_MEMOIZED_VALUES，值将会被计算并存储到字典中。然后，如果带有同样的参数再次调用该函数，结果将会从字典中获取而不是再次计算

这个特性可以很容易的想到与装饰器一起使用。从Python3.3开始，functools模块提供了LRU（least-Recently-Used）cache decorator.

作用This provides the same functionality as the memoization
described here, but with the benefit that it limits the number of entries in the cache,removing the least recently used one when the cache si⁵e reaches its maximum size.The module also provides statistics on cache hits, misses, etc. In my opinion, these are a must-haves when implementing such a cache. There’s no point in using memoization– or any caching technique – if you are unable to meter its usage and usefulness

The AST

  1. AST是Abstract Syntax Tree的缩写。它是一种程序语言源代码的抽象表达方式–树形表示。Python通过解析其源文件建立起自己的AST。
  2. python提供了ast模块来处理AST相关内容
  3. AST用法示例：
    import ast ast.parse <function parse at 0x7f062731d950> ast.parse("x = 42") <_ast.Module object at 0x7f0628a5ad10> ast.dump(ast.parse("x = 42")) "Module(body=[Assign(targets=[Name(id='x',ctx=Store())],value=Num(n=42)])"

ast.parse函数返回一个_ast.Module对象，它是树的根.可以通过ast.dump模块来建立完整的树，如下图所示

  1. AST以一个根元素开始，该元素通常是ast.Module对象。这个对象包含语句或表达式的列表来计算它body属性值。ast.Assign对象代表赋值，在Python语法中映射为“=”，Assign包含target列表和赋给target的值。这个例子中target列表中包含一个对象，ast.Name，代表名为x的变量。值为42
  2. AST可以传递给Python进行编译和计算。compile函数提供了支持
  3. 需要了解的两类对象列表：statement：如assert、assign(=)、augmented assigned(+=,/=)、global、if、return、class、pass、import等。他们全部从ast.stmt继承而来。Expression:lambda、number、yield,name,compare、call.他们从ast.expr继承而来

方法和装饰器

创建装饰器

  1. 装饰器的定义：首先装饰器是一个函数，并且它的参数也是一个函数，返回一个新的函数或修改后的函数
  2. 经过装饰器修饰形成的新函数会丢失原来函数的一些特性，可以使用functools模块解决 `@functools.wraps(f) #f为原函数`
  3. 使用inspect模块获取函数参数，返回值为包含函数名称、函数值的键值对字典形式

类方法、静态方法以及抽象方法的混合使用

  1. 抽象方法的属性并不是一成不变的，我们可以根据需要扩展属性列表，甚至我们可以按我们自己喜欢的方式对其重新定义，只要符合在超类中定义的方法。例如可以将其实现为类方法或静态方法。从Python3开始，允许将@staticmethod、@classmethod放置在@abstractmethod之上

super的真相
通过使用super()函数调用父类中的方法，但实际上super()是一个构造器，当每次调用它时都会实例化一个super对象。它可以接受一个或两个参数，第一个参数是一个类，第二个参数是子类或第一个参数的一个实例

生成器：每次调用next()时返回一个值，直到引发StopIteration异常。写一个生成器就是在所写的函数中包含yield语句。Python将检测到yield的使用，并将函数标记为生成器。当函数执行到yield语句时，通过return语句返回一个值，编译器将会保存一个栈上下文环境，在下一次next被调用时重新使用。可以使用inspect来检查是不是一个生成器。Python3提供了另一个有用的函数，inspect.getgeneratorstate,这个函数可以让我们了解到生成器的状态，如是否等待第一次运行(GEN_CREATED)，是否正在执行（GEN_RUNNING）,是否等待重新使用（GEN_SUSPENDED）,是否完成运行（GEN_CLOSED）.生成器可以帮助我们使用少量的内存消耗来处理大的数据结构。当需要处理大量数据时，生成器可以确保高效的处理.yield有一个不常用的特性，它可以像函数调用一样返回一个值。这样就可以通过调用它的send()方法向生成器传递值

列表推导式：在声明列表的同时定义列表内容 x = [i for i in (1, 2, 3)] 与for循环相比，使用列表推导式是一种更方便迅速地方式

Python包含了大量工具来支持函数式编程，列举如下：

  1. map(function, iterable) applies function to each item in iterable and returns either a list in Python or an iterable map object in Python
  2. filter(function or None, iterable) filters the items in iterable based on the result returned by function, and returns either a list in Python , or better, an iterable filter object in Python
  3. enumerate(iterable[, start]) returns an iterable enumerate object that yields a sequence of tuples, each consisting of an integer index (starting with start, if provided) and the corresponding item in iterable. It’s useful when you need to write code that refers to array indexes.
  4. sorted(iterable, key=None, reverse=False) returns a sorted version of iterable. The key argument allows you to provide a function that returns the value to sort on.
  5. any(iterable) and all(iterable) both return a boolean depending on the values returned by iterable.
  6. zip(iter1 [,iter2 […]]) takes multiple sequences and combines them into tuples. It’s useful when you need to combine a list of keys and a list of values into a dict. Like the other functions described above, it returns a list in Python and an iterable in Python
  7. The itertools module in the Python Standard Library also provides a bunch of useful functions that you’ll want to keep in mind.

Unit testing

对于现在流行的测试驱动开发，作者的观点是“Writing code that is not tested is essentially useless, as there’s no way to conclusively prove that it works.”

  • 基础工具–unnitest包（python的标准类库），介绍了它的一些基本方法如assertEqual、assertTrue,assertFalse,assertGreater,assertGreaterEqual, assertIn,assertIs,assertIsIntance,assertIsNone等等，着重介绍了一下unittest.TestCase.skipTest()的用法–跳过一些不能运行的测试。setUp和tearDown方法分别在类测试方法执行前执行。This is really useful for creating test environments
  • Fixtures–测试夹具由一组unnitest组成，这个组件可以在其他地方重复使用
  • Mocking Mock对象用了模拟真实的应用。如在写一个HTTP客户端时，我们不可能再写一个HTTP server用了返回各个可能值进行测试，也无法满足其他应用场景的测试。如果可以建立起一些模型来模拟特定的场景，然后用这些模型作为环境来测试代码，这样就变得很容易。Mock的作用就在这。通过使用mock，可以模拟以下问题：web服务器返回404错误或者网络问题的出现，可以确保代码返回正确的值或者引发相对应的异常，总之就是代码按照期望的方式工作
  • Scenarios–即场景。很多情况下，需要在不同版本中进行测试；或者在不同驱动上进行测试。对不同驱动的实现情况，Ceilmeter提供了一个抽象类，然后不同驱动去实现这个类然后注册成driver。这样在测试时就需要对各个驱动进行测试。testscenarios提供了一种简单的方式在不同情景中运行一组测试。
  • Test streaming and parallelism：在进行测试时，对他们的运行过程进行分析显得非常有用。sububit对测试结果提供了流的处理。它允许对测试结果进行合并、记录和存档。它也可以自动检测出哪些测试在运行。testrepository用来处理大量测试的结果流化。提供了testr程序，来处理一个测试的数据库:例如记录自从上次提交后哪个测试失败了。并行运行测试的方法testr run --parallel
  • Coverage：覆盖测试。code coverage使用代码分析工具和追踪工具来确定哪些行代码已经执行过了，哪些还没有。python模块coverage提供了这些功能。一旦安装成功后，可以使用命令行与coverage程序交互
  • Using virtualenv with tox：tox的目标是使python的测试变得自动化和标准化。tox可以完成部署虚拟环境、安装setup工具，安装所有应用/类库/单元测试所需的依赖。当运行时，tox将创建虚拟环境，安装新的依赖并且运行测试

openstack-cinder bug的一次重现与修复

發表於 2015-12-02   |   分類於 openstack
  • bug描述：openstack在创建一个project A时，默认分配的volume quota为10，将该quota完全分给其子项目B，然后再在A中创建volume时可以创建成功。正常情况来说不应允许新的volume创建，因为quota全部分配给了B，这就是bug的具体体现
  • 实验环境 ：centos7 openstack L版本(devstack安装)
  • bug恢复步骤 ：
    1. openstack project create A
    2. grant the role admin to the user in this project
      openstack role add admin --project A --user admin
    3. create a subproject of A
      openstack project create B --property parent_id=<project id of A>
    4. grant the role admin to the user in this project.
      openstack role add admin --project B --user admin
    5. . openrc admin A
    6. allocate all volume quota of project A to project B openstack quota set <project id of B> --volumes 10
    7. create a volume in project A, it should not be allowed since it was allocated all the quota of project A to project Bopenstack
      volume create --size 1 test_volume
      在第七步时，应该不允许创建新的volume。但是仍然可以创建成功。原因是没有将父project A 分配出去的quota计算在内。
  • 修复方法：修改cinder/db/sqlalchemy/api.py文件，具体修改 ：获取项目分配出去的quota–allocated；在使用量中添加allocated
  • 重启cinder服务：操作见stackoverflow,我按这个方法做没有成功，但我觉得这个方法是正确的，只是我操作的问题，于是就把cinder部分git下来，然后修改相应文件，最后再进行安装
  • bug修复后的状态：在执行第7步时，会提示“maximum number of volumes allowd (10) exceeded for quota ‘volume’”,并拒绝请求的执行
  • 踩过的坑与总结：我觉得这个bug比较好理解，解决办法也比较简单，但是我大部分时间与精力都浪费在了环境上。刚开始时我用了rdo安装了K、L版本，发现使用rdo的安装不能重现该bug，并且第6条命令只有L版本支持。于是再用devstack进行安装L版本，中间也出现了不少问题，安装成功后进行bug修复，在重启服务时，由于没有成功重启服务，加上其他问题，导致环境崩溃。最后选择git代码，修改文件然后进行安装，最后成功修复bug。从最初的bug重现到修复，前前后后大概用了一个多月。所以在进行bug重现的时候一定要把环境配置弄清楚
