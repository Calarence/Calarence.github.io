全栈式JavaScript

發表於 2015-08-05   |   分類於 技术杂文

最近学习了一段时间的MEAN框架，并用它做了两个项目，其中没有用到M，即mogodb，而是是使用了mysql代替。一开始接触的时候，对使用js写服务器端感到比较惊奇，因为我对js的认识还停留在用于写前端弹出提示框。但随着对MEAN框架的接触，渐渐明白并开始接受使用开发服务器端。

伯乐在线的这篇文章详细的介绍了js，从为什么选择js，端到端的交互，一直到js的测试。我把它的图片摘出来，用它来描述MEAN框架各部分间的关系。其中的一些与图片上的解释相似
MEAN即Mongodb、Express、Angularjs、Nodejs四部分的首字母

  1. 标注1是交互界面，可使用bootstrap快速搭建简洁风格的界面
  2. 标注2对应MEAN的A部分，即angularjs,angularjs实现了对DOM的封装，使我们不必省去了直接操作dom元素的麻烦。此外angularjs还可以构建单页面应用
  3. 标注3对应的是angularjs与后端E，即express的交互—使用HTTP请求
  4. 标注4说明的是数据文件传输格式为json
  5. 标注5描述的是使用nodejs与mongodb数据库进行交互，mongodb使用json格式存储数据其中基于nodejs的框架Express提供了与mongodb、mysql等数据库交互的中间件

  6. 标注6描述的是js对象，即model层，基于面向对象的js编程
